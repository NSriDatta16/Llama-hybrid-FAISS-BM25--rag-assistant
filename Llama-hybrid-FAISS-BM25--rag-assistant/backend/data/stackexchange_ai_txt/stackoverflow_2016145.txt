I ran into this issue also. I use a mixture of Zend_Locale and a custom xml file. Zend_Locale is used to generate the country list: $locale = new Zend_Locale(Zend_Locale::BROWSER); $countries = $locale->getTranslationList('Territory', Zend_Locale::BROWSER, 2); asort($countries, SORT_LOCALE_STRING); // unset invalid countries unset($countries['SU'], $countries['ZZ'], $countries['IM'], $countries['JE'], $countries['VD']); Then my custom XML file looks like this: Andorra Parròquia de Canillo Parròquia d'Encamp Parròquia de la Massana Where each child element of is a 2 letter country code corresponding to the key value of $countries . I then use ajax/php/xpath to repopulate the state/province drop down any time the country drop down value is changed. Feel free to download my XML file here: http://gregan.org/provinces.xml I don't remember where I got the original data, nor do I guarantee it's validity
