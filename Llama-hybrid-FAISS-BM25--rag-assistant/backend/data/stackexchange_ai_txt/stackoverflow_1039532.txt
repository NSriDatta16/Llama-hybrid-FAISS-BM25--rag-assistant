I was getting a similar issue, to stop the error appearing you want to catch the IOErrorEvent.IO_Error _loader.contentLoaderInfo.addEventListener(IOErrorEvent.IO_ERROR, ioError_handler, false, 0, true); public function ioError_handler(event:IOErrorEvent):void { Alert.show(event.toString()); } Whilst this got rid of the error, some of the images where not loading in Opera. In the end I got this to work by changing the image handler code (asp.net c#) to context.Response.ClearContent(); context.Response.ClearHeaders(); context.Response.ContentType = "image/jpg"; context.Response.Cache.SetCacheability(HttpCacheability.NoCache); context.Response.Cache.SetNoStore(); context.Response.Cache.SetExpires(DateTime.MinValue); context.Response.BufferOutput = false; context.Response.BinaryWrite(photo); before when I tried to view the image in the browser the photo would load, but it wouldn't finish loading in Opera, hence not firing the load complete event. Hope this helps, (even if it is in a different language)
