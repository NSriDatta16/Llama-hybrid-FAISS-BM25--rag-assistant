One idea would be to create a custom handler, then map to the expected requests. Here is a link for more information on handlers. Pay special attention to the configuration that goes into your web.config for each handler. The path that is defined does not have to be a path that exists. One nice trick is to place your handler configuration within a grouping within your web.config. That way you can have various "virtual" paths within your application for handling different scenarios.
