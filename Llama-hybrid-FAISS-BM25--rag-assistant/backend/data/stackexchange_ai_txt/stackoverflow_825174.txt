The hash character is meant for client side navigation. Anything after the # is not submitted to the server. From the wikipedia article : The fragment identifier functions differently than the rest of the URI: namely, its processing is exclusively client-side with no participation from the server. When an agent (such as a Web browser) requests a resource from a Web server, the agent sends the URI to the server, but does not send the fragment. Its technical name is Fragment Identifier
