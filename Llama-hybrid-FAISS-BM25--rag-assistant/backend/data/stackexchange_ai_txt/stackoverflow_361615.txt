Since http is connectionless, the only way to associate the retrieval with the query would be with sessions. THen you'd have all these answers waiting around for someone to claim them, and no way to know if the connection (that doesn't exist) has been broken. In a web page, it's pretty much use-it-or-lose-it. Some of the other answers might work with a lot of effort, but I don't get the sense that you're looking for an edge-case, high-tech option.
