I guess this can be measured quite easily. Get the amount of resources used by the system before creating a thread Create a thread with default system values (default heap size and others) Get the amount of resources after creating a thread and make the difference (with step 1). Note that some threads need to be specified different values than the default ones. You can try and find an average memory use by creating various number of threads (step 2). The memory allocated by the OS when creating a thread consists of threads local data: TCB TLS ... From wikipedia : "Threads do not own resources except for a stack, a copy of the registers including the program counter, and thread-local storage (if any)."
