I've been told that there is some overhead in using the Java try-catch mechanism. Absolutely. And there's overhead to method calls, too. But you shouldn't put all your code in one method. Not to toot the premature optimization horn, but the focus should be on ease of reading, organization, etc. Language constructs rarely impact performance as much as system organization and choice of algorithms. To me, the first is easiest to read.
