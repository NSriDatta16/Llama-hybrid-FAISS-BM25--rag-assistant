No. replaceChild would remove one from the DOM. You want insertBefore var a = document.getElementById('item1'); var b = document.getElementById('item2'); a.parentNode.insertBefore(b, a); Of course, this would only swap them if they were next to each other in the first place. You could leverage replaceChild if you wanted to swap ones which were not next to each other. var a = document.getElementById('item1'); var b = document.getElementById('item3'); var after_b = b.nextSibling; var p = a.parentNode; p.replaceChild(b,a); if (after_b) { p.replaceChild(a, after_b); } else { p.appendChild(a); }
