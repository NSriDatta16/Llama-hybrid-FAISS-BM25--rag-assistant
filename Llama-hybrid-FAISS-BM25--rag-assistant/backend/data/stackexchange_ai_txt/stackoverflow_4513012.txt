Or, one could use an existing standard. ISO 5218 covers four codes: 0 = Not Known 1 = Male 2 = Female 9 = Not applicable (lawful person such as corporation, organization etc) ISO 5218 is a legal encoding and does not apply for medical/biological aspect. Obviously, a reference table containing those codes should use the natural key (as per above list), and not a syntetic key. Joe Celko's Data Measurements And Standards in SQL is a great (albeit boring) read.
