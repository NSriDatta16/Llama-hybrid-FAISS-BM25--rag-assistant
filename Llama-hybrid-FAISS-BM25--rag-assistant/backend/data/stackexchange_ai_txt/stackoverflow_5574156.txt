The only more elegant, and more convoluted, solution would be to use a data-driven approach: Have a .plist file containing what you consider to be your default settings. If the program detects that the user defaults is empty, it will load this default plist, and commit it to NSUserDefaults. Using this method your code is not responsible for building the objects. However, if you are trying to accomplish a schema-upgrade, you're going to need to go back to the code.
