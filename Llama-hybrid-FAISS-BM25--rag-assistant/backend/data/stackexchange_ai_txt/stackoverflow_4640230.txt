1) All cocos2d-iphone code is handled within a single main game loop, ccTouchBegan is called by the Director when an iPhone SDK touch event occurs. 2) All "events" are handled before the frame is processed, rendering is the last thing that is done. 3) ... There are no multi-threading issues in Cocos2D.
