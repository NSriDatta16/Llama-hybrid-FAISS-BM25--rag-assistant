You should write unit tests where each test reveals some existing behavior of the already implemented code. One additional thing could be to have 100% code coverage of your xml parsing library before you start refactoring it. I could not comment much without looking at the public interface of your xml library. On a side note, Unit Tests acts as a safety net which helps you to catch any breacking functional changes done during refactoring. Unit tests will enable you for refactoring.
