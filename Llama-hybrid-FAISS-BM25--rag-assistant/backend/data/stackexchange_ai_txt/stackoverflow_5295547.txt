The main reason is your code will be ugly if you merge those php ( business logic ) with the html ( presentation ) together, which in turn become hard to read and become hard to maintain . It won't be a problem if your web application is a simple one. But if it's a enterprise scale project, maintaining this merged code will be a nightmare for anyone. Beside, the kind of programming you do for each part and sometimes the language you do it in are different. Separating the two allows one to use the best tools for that particular part. source: http://www.paragoncorporation.com/ArticleDetail.aspx?ArticleID=21
