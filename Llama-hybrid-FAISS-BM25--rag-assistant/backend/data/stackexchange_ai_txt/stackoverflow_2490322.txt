You would probably find something like the following would perform better. The filter approach will be forced to iterate through each subscriber, while the NonEmpty() function can take advantage of optimizations in the storage engine. select non empty{ [Measures].[Count], [Measures].[Cost], [Measures].[Topup Value] } on columns, { NonEmtpy( [Subscriber].[Subscriber Key].Children, ( [Measures].[Topups Count], [Topup Date].[Calendar].[Month Name].&[2010]&[3] ) ) } on rows from [Calls] ;
