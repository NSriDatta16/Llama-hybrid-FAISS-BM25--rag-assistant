If the functions and/or classes contained in header (say, A.h) depend on functions and/or classes defined in another header (say, B.h), I prefer to include the latter in the first, rather to force the users of the first one to include both in a particular order. Yes: // A.h #pragma once // or the #ifndef trick #include "B.h" // A.cpp #include "A.h" No: // A.h #pragma once // or the #ifndef trick //#include "B.h" // A.cpp #include "B.h" #include "A.h"
