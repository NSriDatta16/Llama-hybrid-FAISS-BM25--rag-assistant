One is safer than the other. When you check if a file exists, nothing guarantees that file is there further down the execution of the method. If you surround it with a Try / Catch block, you can elegantly catch the glitch, act accordingly and wrap up any thing you want to in the Finally block. For example, closing the stream to the file.
