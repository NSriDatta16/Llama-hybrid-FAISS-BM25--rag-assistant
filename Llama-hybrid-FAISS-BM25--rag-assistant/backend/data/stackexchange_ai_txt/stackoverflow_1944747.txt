Perhaps consider using SQL Server Agent to schedule the query as job? This is especially worthwhile if the job needs to run on a regular schedule or only very rarely. If absolutely necessary, you can also use use a SQL script to schedule the job on demand from your webapp. Just ensure the agent service is running, otherwise your job won't start.
