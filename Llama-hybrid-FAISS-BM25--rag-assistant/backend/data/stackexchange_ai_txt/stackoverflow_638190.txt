If it is a c# DLL then you can use ildasm (Program Files\Microsoft SDKs\Windows\v6.0A\bin\ildasm.exe) to find out this information. 1) Drag DLL into ILDASM 2) Dbl-Click on the MANIFEST 3) Look for: // .custom instance void [mscorlib]System.Diagnostics.DebuggableAttribute::.ctor(valuetype [mscorlib]System.Diagnostics.DebuggableAttribute/DebuggingModes) = ( 01 nn nn nn nn nn nn nn ) 4) For DEBUG DLLs it will be ( 01 00 07 01 00 00 00 00 ) and for release (01 00 02 00 00 00 00 00) or (01 00 03 00 00 00 00) Let me know if you need any further info! BTW This is obviously the non programmatic solution.
