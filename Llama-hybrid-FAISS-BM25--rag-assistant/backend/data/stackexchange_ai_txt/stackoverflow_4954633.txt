It's incredibly likely you've considered this and you're likely only looking for a solution using the existing undoManager , but just in case: Since you're inserting objects and not updating existing ones, you do have the power to tag them with a transaction id as each batch is imported, deleting them in a background thread in the case of an undo. A simple incremented NSNumber is sufficient for the tag. Inelegant, but workable.
