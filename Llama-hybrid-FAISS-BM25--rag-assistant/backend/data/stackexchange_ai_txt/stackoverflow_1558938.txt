If by blob you mean PostgreSQL bytea, then you can just use select substring(bytea_column from ? for ?) . It is fast when data blob is not compressed - you should prevent automatic compression of bytea using alter table tablename alter column bytea_column set storage external . Only rows inserted after alter table will not be compressed so you should do it on empty table or delete and reinsert all data after this command. I'm using it to retrieve blobs (binary file data like DOC or PDF) from database in chunks. Works well.
