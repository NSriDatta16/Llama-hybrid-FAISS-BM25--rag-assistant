Use the regular membership as described in the other answers. Then leverage the Profile system so that each user can view/edit their info when logged in (per the question). CAVEAT: ASP.NET profile system only works out of the box with the Website project template. If you want to use the Web Application project template, then follow the steps here: ASP.NET: Web Site versus Web Application Project When you have the profiles up and running, the profile data can be stored in session objects while the user is logged in.
