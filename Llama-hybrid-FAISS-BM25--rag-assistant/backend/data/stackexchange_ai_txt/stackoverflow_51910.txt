Some commands i use in my code : \todo { paragraph describing what is to be done } Useful to keep track of todos, a page will be created in final documentation containing your todo list. \c Displays the argument using a typewriter font. Use this to refer to a word of code. I would use it before "TRUE" and "FALSE" in your example. \a , \warning , \see : see http://www.stack.nl/~dimitri/doxygen/commands.html#cmdc for description
