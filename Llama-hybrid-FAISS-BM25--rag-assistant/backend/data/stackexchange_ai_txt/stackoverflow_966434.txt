The answer is explained here: http://www.w3.org/Protocols/rfc2616/rfc2616-sec14.html#sec14.9.4 When used by user agents, its aim is to get intermediate caches to revalidate the response - so it's not for the server to deal with. In server responses, max-age tells the client (and intermediate caches) how long to cache the response for.
