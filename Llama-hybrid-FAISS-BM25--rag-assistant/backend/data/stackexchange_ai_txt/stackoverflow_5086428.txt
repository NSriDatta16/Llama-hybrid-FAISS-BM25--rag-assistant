If you just need to do some one-off number crunching, a spreadsheet is far and away your best tool. It's trivial to spit out a simple CSV file from C#, which you can then load up in Excel (or whatever): class Program { static void Main(string[] args) { using (StreamWriter sw = new StreamWriter("output.csv", false, Encoding.ASCII)) { WriteCsvLine(sw, new List () { "Name", "Length", "LastWrite" }); DirectoryInfo di = new DirectoryInfo("."); foreach (FileInfo fi in di.GetFiles("*.mp3", SearchOption.AllDirectories)) { List columns = new List (); columns.Add(fi.Name.Replace(",", " ")); columns.Add(fi.Length.ToString()); columns.Add(fi.LastWriteTime.Ticks.ToString()); WriteCsvLine(sw, columns); } } } static void WriteCsvLine(StreamWriter sw, List columns) { sw.WriteLine(string.Join(",", columns.ToArray())); } } Then you can just 'start excel output.csv' and use functions like "=MEDIAN(B:B)", "=AVERAGE(B:B)", "=STDEV(B:B)". You get charts, histograms (if you install the analysis pack), etc. The above doesn't handle everything; generalized CSV files are more complex than you might think. But it's "good enough" for much of the analysis I do.
