Not 100% what causes this, might be chrome's scrollbars rendering as html. I'm afraid you'll need to initialize a draggable with the handle option specified. $( "#id" ).draggable({ handle: '.yourhandleclass' }); It might also be possible to use the 'Cancel' option specified. (Prevents dragging from starting on specified elements.) Like this: $( "#id" ).draggable({ cancel: '.replacethiswithchromescrollbarselector' });
