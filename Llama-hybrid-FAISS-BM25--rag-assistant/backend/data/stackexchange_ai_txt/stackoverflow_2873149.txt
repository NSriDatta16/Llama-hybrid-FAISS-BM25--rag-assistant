First, a better solution would to be use a richer language for expressing links to the text-only version. It exists! RFC 5147: URI Fragment Identifiers for the text/plain Media Type . For instance: http://www.ietf.org/rfc/rfc2518.txt#line=1000 It has a (very limited) implementation and a (better?) one in the browser Amaya. For the HTML (unofficial) versions, there is the possibiity you request at the excellent tools.ietf.org . For instance: https://www.rfc-editor.org/rfc/rfc2518#section-9.8
