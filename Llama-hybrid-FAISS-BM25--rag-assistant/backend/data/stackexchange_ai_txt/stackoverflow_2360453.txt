And the following css: #wrap {width: 70%; margin: 1em auto; overflow: hidden; overflow-x: visible; } #content {width: 90%; margin: 0 auto; -moz-box-shadow: 0 0 1em #ccc; -webkit-box-shadow: 0 0 1em #ccc; } #content p {overflow-y: hidden; padding: 0.5em 0; } (Live demo located here: http://davidrhysthomas.co.uk/so/shadows.html .) The fragility creeps in if you add margins to the contained elements (especially the elements, which is why I used padding instead). But, pretty much, apply the -moz-box-shadow (and/or -webkit-box-shadow ) to the #content div, and use the #wrap div to clip the shadow, using overflow-y: hidden; this, of course, makes it even more fragile due to the number of browsers that respect overflow-y . On the other hand, the browsers that interpret the box-shadow more or less certainly deal appropriately with overflow-y .
