There are a number of session state modes in ASP.NET you can use (which can be configured in web.config) apart from the default "In Proc": StateServer mode, which stores session state in a separate process called the ASP.NET state service. This ensures that session state is preserved if the Web application is restarted and also makes session state available to multiple Web servers in a Web farm. SQLServer mode stores session state in a SQL Server database. This ensures that session state is preserved if the Web application is restarted and also makes session state available to multiple Web servers in a Web farm. Custom mode, which enables you to specify a custom storage provider. See ASP.NET Session-State Modes on MSDN . I haven't tested this for HTTPS, but hopefully StateServer or SQLServer should facilitate this.
