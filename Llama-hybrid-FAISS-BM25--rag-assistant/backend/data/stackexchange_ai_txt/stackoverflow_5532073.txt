The string is already HTML, and you are working in Rails, so I'm not sure that you really need to boil it down further. While the HTML numeric character references in your example string do have both ASCII and Unicode equivalents, anything more complicated like will cause trouble. However, for your example string, there is a way. You can use an HTML parser to parse the string as a fragment. require 'nokogiri' s1 = 'How&#39;s life?' s2 = Nokogiri::HTML.fragment(s1).to_s puts s2
