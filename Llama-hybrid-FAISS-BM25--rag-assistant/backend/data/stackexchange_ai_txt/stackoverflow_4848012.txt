Seems like a good usecase for enums . So take out the interface and replace it with an enum. Since it is a collection of constants enum fits the bill nicely. Moreover, enums are one of the most efficient ways to implement a Singleton in Java. Instead of duplicating answers, take a look at these good relevant questions. Java Enum Singleton Efficient way to implement singleton pattern in Java Alternatively as Pangea mentioned you can do static imports . I think both approaches are fine but enums in my opinion will be a better placeholder for organizing your unrelated constants in relevant meaningful groups.
