The execution plan is stored in the cache as others have pointed out and even table data is stored in the cache an will stay there as long as possible. The compile time is not the issue in your case. it may be: the stored procedure is using a different execution plan your indexes are fragmented the statistics are out of date, causing an incorrect execution plan you do not have enough memory try to use the followin code Before you execute your stored procedure SET STATISTICS IO ON and look to see if your procedure uses physical read or logical reads. If it is logical it uses the cache to get the data
