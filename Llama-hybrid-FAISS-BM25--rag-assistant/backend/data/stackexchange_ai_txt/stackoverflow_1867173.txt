If I understand your requirements correctly, a Silverlight 3 out-of-browser app would give you the best of both worlds: easy to install easy to push out to users can be launched from start bar/desktop shortcut can run offline (they call it detached state), so if your server is down you can notify users nicely through the local app, rather than them browsing to the online SL url and getting a 404. local storage easy to push application updates out
