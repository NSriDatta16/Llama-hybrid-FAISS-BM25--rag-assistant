B, yes IF the cost of spawning is zero then legacy CGI would be pretty good. So if you don't have a lot of hits plain old CGI is fine, run with it. The point of fast cgi is doing things that benefit from a lot of persistent storage, or structures that have to be built BEFORE you can get your work done, like running queries against large databases, where you want to leave the DB libraries in memory instead of having to reload the whole shebang every time you want to run a query. It matters when you have LOTS OF HITS.
