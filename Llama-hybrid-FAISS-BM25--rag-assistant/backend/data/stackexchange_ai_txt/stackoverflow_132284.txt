I'm not aware of a tool that does this well but I've seen a variety of homegrown solutions. The common thread of these is to minimise the binary data under version control and maximise textual data to leverage the power of conventional scc systems. To do this: Treat the workbook like any other application. Seperate logic, config and data. Separate code from the workbook. Build the UI programmatically. Write a build script to reconstruct the workbook.
