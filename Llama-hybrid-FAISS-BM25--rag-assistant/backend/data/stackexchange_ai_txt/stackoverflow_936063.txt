Will it not be easier and clean just to pass the current controller as an argument to the model method call? Like following: def delete_cascade(controller) self.categories.each do |c| c.delete_cascade(controller) controller.expire_fragment(%r{article_manager/list/#{c.id}.*}) end PtSection.delete(self.id) controller.expire_fragment(%r{category_manager/list/#{self.id}.*}) end You can access all public methods and properties of the controller from within model. As long as you do not modify the state of the controller, it should be fine.
