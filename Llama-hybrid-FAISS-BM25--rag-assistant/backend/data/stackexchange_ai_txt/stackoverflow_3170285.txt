It looks like from the Ecmascript spec , the JSON object shouldn't be treated as a valid Javascript program: "Note that an ExpressionStatement cannot start with an opening curly brace because that might make it ambiguous with a Block. So assuming that all browser implement this correctly, a response like { name: 'Puff the Dragon', cc: 'Credit Card #' } won't be executed as valid Javascript. However expressions like ({name: 'Puff the Dragon', cc: 'Credit Card #' }) and {['Puff the Dragon', 'Credit Card #']} will.
