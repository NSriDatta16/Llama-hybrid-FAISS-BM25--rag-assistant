Fixed it! My new code is listed below. The issue was IE's handling of Flash parameters (wouldn't perform an async refresh of the params for the object). It was working fine in FF because it was embedding the video, and not handling it as an object. New Javascript function display_youtube(new_url) { $('#object_url').replaceWith(' '); $('#embed_url').replaceWith(' '); $('#shade').css('display', 'block'); $('#youtube_player').css('display', 'block'); $('#exit_youtube').css('display', 'block'); } function exit_youtube() { $('#object_url').replaceWith(' '); $('#embed_url').replaceWith(' '); $('#shade').css('display', 'none'); $('#youtube_player').css('display', 'none'); $('#exit_youtube').css('display', 'none'); } New HTML I think the trick here was to force IE to rely on instructions from jQuery before it could perform any actions whatsoever on the object, in effect preventing IE from caching any parameters from the start. FTW!
