That's grouping adjacents para elements. The problem is in Conflict Resolution for Template Rules : because both para matching rules have the same import precedence and the same calculated priority, error recovery mechanism could lead to apply last one. You need to explicitly define a @priority on the "first para child" rule like: With such modification, the output is: Main Section Here is some text that is a child of a main section. Some more text. When a section has subsections, it should not have loose paragraphs before the first sub section. Those loose paras should be placed inside a comment element. This is my subsection Text that is inside of the sub-section And some more sub section text.
