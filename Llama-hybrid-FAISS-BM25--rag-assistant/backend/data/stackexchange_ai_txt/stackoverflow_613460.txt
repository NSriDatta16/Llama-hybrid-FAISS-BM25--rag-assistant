If you know to call Refresh(), why not just go ahead and refresh the UI at that point anyway? PropertyChanging and PropertyChanged are invoked by invoking the setter on a LINQtoSQL DBML-generated entity class. Calling Refresh() does not do that: [Column(Storage="_DisplayName", DbType="VarChar(50) NOT NULL", CanBeNull=false)] public string DisplayName { get { return this._DisplayName; } set { if ((this._DisplayName != value)) { this.OnDisplayNameChanging(value); this.SendPropertyChanging(); this._DisplayName = value; this.SendPropertyChanged("DisplayName"); this.OnDisplayNameChanged(); } } }
