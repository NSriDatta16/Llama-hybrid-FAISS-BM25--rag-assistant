Seems you should use RrdDb("/path/to/my/file") instead. From Javadocs: RrdDb(java.lang.String path): Constructor used to open already existing RRD in R/W mode, with a default storage (backend) type (file on the disk). And also: RrdDb(RrdDef rrdDef): Constructor used to create new RRD object from the definition.
