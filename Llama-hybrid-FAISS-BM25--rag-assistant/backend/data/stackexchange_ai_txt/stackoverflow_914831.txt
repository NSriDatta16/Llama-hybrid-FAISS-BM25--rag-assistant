maybe the link above may help you out, why these strange things are happening. "Pay attention if you run .NET Framework 3.5 SP1 on a non-English OS! If you run the .NET Framework 3.5 SP1 on a non-English OS, it will attempt to install the .NET Framework 3.5 SP1 language pack in the language that matches the OS language by default. As the language pack is not a part of the full package, the .NET Framework 3.5 SP1 installer will attempt to download it and install it for the user behind the scenes on non-English OS's".
