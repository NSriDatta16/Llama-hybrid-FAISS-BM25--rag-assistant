Let me start by saying that evaluating a project based on criteria like this is just as goofy as evaluating a programmer based on the same thing. I know that there are studies that show that there is a clear correlation between number of lines of code and nuber of code defects. In my opinion that is simply a matter of increased scale. Having said that, if your overlords...err...I mean management requires you to come up with something here are some relatively easy measurements to make: Total number of nodes Number of node types Average attributes per node Greatest level of nesting
