No, this wouldn't work. Authentication to Windows Azure storage involves signing the headers of the request. (See http://msdn.microsoft.com/en-us/library/dd179428.aspx .) Note that the .NET StorageClient library that ships with the SDK is redistributable, so you can just add a reference to that and do (from memory): CloudStorageAccount.Parse(" ") .CreateCloudBlobClient() .GetBlobReference("mycontainer/myblob") .UploadByteArray(data);
