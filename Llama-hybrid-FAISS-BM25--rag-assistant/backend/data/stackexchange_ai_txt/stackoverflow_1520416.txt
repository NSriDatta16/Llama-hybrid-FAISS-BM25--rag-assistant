I believe it's referring to byref arguments not functions . For example this: function doStuff(&$value1, &$value2) { ... } is an acceptable use of byref because the doStuff() function has to return 2 values. If it only doStuff() only needed to affect one value, it would be more elegant to have the function return it, by value, of course.
