I would go the other way around. And I mean by that, not making queries and database connections within Excel, but using some sort of Web Application to let users (through wizards) generate data they need, and export them to Excel to work. That way you will have the following benefits: No DB connections (and probably passwords) in your Excel files No distribution problem of Excel files with new queries, views, etc. Centralized approach to data retrieval Excel for users used to it
