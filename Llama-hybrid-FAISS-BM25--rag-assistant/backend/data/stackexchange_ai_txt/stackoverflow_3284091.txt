It is possible that the benefits sought by the the person who chose to create 110 projects can be achieved with 110 namespaces in one .dll. The large number of dll's makes it more obvious when dependencies start to creep (1 dll references 22 others), but that is also true with namespaces (if you don't have a using statement, then it there is build in discouragement to reference lots of other namespaces. Also when you merge assemblies, you lose "internal", and may want to make some methods less visible than they were, which is a good thing anyhow.
