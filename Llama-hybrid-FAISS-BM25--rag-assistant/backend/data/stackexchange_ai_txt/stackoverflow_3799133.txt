You need: Whenever you declare a param or variable without select attribute and with some content template, the variable or parameter will be of type Result Tree Fragment. Whenever you output an attribute node, it's an error if you don't output it before any other node type of some element's content template. Error recovery mechanism could be to silently output nothing. In XSLT 2.0 the error is rised. Note : I'm ussing &apos; entity because you have wrapped the text with ' , otherwise it's no needed.
