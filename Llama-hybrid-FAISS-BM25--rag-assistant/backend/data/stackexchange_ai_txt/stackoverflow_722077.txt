When talking about separation of data files, the word disk can be interchanged with LUN's. It sounds though as if you are only using a single LUN i.e. presenting your entire RAID 5 array as a single partition. Do you have 16 CPU's or 16 cores, for example 4 quad core CPU's? You should only really look to create a separate data file for each 'true' CPU in my opinion. I suggest you read the following articles regarding the tempdb database before going ahead with any changes. Paying particular attention "Should I create multiple tempdb database files?".
