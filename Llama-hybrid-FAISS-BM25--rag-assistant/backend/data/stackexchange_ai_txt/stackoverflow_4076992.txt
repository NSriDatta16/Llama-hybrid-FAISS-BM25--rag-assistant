You are correct, these parameters only required when monitoring remote application via JMX. When monitoring locally, you can skip them. Obviously the process needs to be started by the same user. I think there are some exceptions to this rule, see http://download.oracle.com/javase/1.5.0/docs/guide/management/agent.html#local Q:"If the port number is not required, how does the client know who to contact Tomcat?" A: jconsole, jps and the likes simply looks for any Java processes that are running on the system and owned by the same user. Read here: http://download.oracle.com/javase/6/docs/technotes/guides/management/jconsole.html (chapter about attachable applications).
