The SQLite Query Planner 4.0 The LIKE optimization Terms that are composed of the LIKE or GLOB operator can sometimes be used to constrain indices. There are many conditions on this use: The left-hand side of the LIKE or GLOB operator must be the name of an indexed column. The right-hand side of the LIKE or GLOB must be a string literal that does not begin with a wildcard character. [...] Update: by default case_sensitive_like mode is OFF. Turning it ON might make it behave like = . PRAGMA case_sensitive_like = ON; -- OFF SELECT * FROM your_table WHERE field LIKE '...'
