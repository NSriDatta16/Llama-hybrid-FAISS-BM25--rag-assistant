Either: Use RAISERROR (severity 10) to fire a SQL agent alert and job. Load a separate table that is polled periodically by a separate mail handling process. (as HLGEM suggested) Use a stored procedure to send the message and write to the table. Each solution decouples the transactional trigger from a potentially long messaging call.
