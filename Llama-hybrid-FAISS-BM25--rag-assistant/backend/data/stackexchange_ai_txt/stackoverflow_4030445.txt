The foreign key is in the Order table. Hence i instinctively always drag from order (table with the foreign key ) to customer (the table having the primary key / unique key ). However, the only reason why it would matters which way you drag the arrow is based on whatever the tool you are doing the modeling allows you to do In the end, what you are doing is creating a foreign key relationship from Order table to Customer Table on the column CustomerId
