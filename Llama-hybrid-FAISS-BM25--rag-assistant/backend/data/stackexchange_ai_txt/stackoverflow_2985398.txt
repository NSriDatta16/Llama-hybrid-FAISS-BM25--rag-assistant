I'd initialise the WHERE clause with 1=1 so everything after that is concatenation with 'AND' and it's never NULL DECLARE @whereClause as nvarchar(1024) SET @whereClause = '1=1' IF @hasSpouse > -1 BEGIN IF @hasSpouse = 0 SET @whereClause = @whereClause + ' AND p.[HasSpouse]=0' ELSE SET @whereClause = @whereClause + ' AND (p.[HasSpouse]=1 OR p.[HasSpouse] IS NULL)' END -- Dynamically add the next filter if necessary IF @isVegan > -1 BEGIN IF @isVegan = 0 SET @whereClause = @whereClause + ' AND c.[IsVegan]=0' ELSE SET @whereClause = @whereClause + ' AND (c.[IsVegan]=1 OR c.[IsVegan] IS NULL)' END PRINT @whereClause BTW, parenthesis around conditions are not needed in SQL.
