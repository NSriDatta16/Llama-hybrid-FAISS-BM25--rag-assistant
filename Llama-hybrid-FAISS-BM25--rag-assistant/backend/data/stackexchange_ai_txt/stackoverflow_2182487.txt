Because a property is a method. It is a language construct responding to a pattern of encapsulating the setting and retrieval of a private field through a set of methods. It is functionally equivalent to this: class Foo { private int _bar; public int GetBar( ) { return _bar; } public void SetBar( ) { _bar = value; } }
