Why isn't MVC binding my nested collection?

If got a couple of linqTOsql objects that I am trying to edit through a form. First, there's the Stream class: [Table] public class Stream { [HiddenInput(DisplayValue = false)] [Column(IsPrimaryKey = true, IsDbGenerated = true, AutoSync = AutoSync.OnInsert)] public long StreamID { get; set; } /*other properties removes for brevity*/ // relationship (one stream to many Stream2FieldTypes) // uses EntitySet and OtherKey for the FK in Stream2FieldTypes private EntitySet _Stream2FieldTypes; [System.Data.Linq.Mapping.Association(Storage = "_Stream2FieldTypes", OtherKey = "StreamID")] public EntitySet Stream2FieldTypes { get { return this._Stream2FieldTypes; } set { this._Stream2FieldTypes.Assign(value); } } } Then there's the Stream2FieldTypes class: [Table] public class Stream2FieldTypes { [Column(IsPrimaryKey = true, IsDbGenerated = true, AutoSync = AutoSync.OnInsert)] public long s2fID { get; set; } /*other properties removed for brevity*/ [Column] public long StreamID { get; set; } // FK // relationship (one Stream2FieldTypes to Streams) private EntitySet _Stream; [Association(Storage = "_Stream", ThisKey = "StreamID")] public EntitySet Stream { get { return this._Stream; } set { this._Stream.Assign(value); } } } I use the linqTOsql repository to get these objects and send them to the view like this: public ActionResult StreamEdit(long id) { Genesis.Domain.Entities.Stream stream = genesisRepository.Streams.FirstOrDefault(x => x.StreamID == id); return View(stream); } Everything get's sent to the view without a problem... But when the form on the view is submitted, the Stream.Stream2FieldTypes gets lost for some reason. I catch the form submission with this action: [HttpPost] public ActionResult StreamEdit( long id, Genesis.Domain.Entities.Stream form) { return View(form); } And the view is without the Stream2FieldTypes collection. The view is writing the Stream2FieldTypes like this: Label: x.Stream2FieldTypes[i].s2fLabel, new { style="width:100px" })%> Required: x.Stream2FieldTypes[i].s2fIsRequired)%> Field Type ID": x.Stream2FieldTypes[i].FieldTypeID, new { style="width:20px;" })%> Stream ID: x.Stream2FieldTypes[i].StreamID) %> Why isn't the default model binder in mvc correctly binding the nested collection?
