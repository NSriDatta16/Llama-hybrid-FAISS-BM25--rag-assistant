All Isolation levels, including snapshot, affect only reads. SELECTs from stageTable will not see uncommited inserts, nor it will block. I'm not sure that solves your problem of throwing everything into the stageTable without any regard for ownership. What happens when the transaction finally commits, the stageTable is left with all the intermediate results ready to be read by the next transaction? Perhaps you should use a temporary #stageTable that will ensure a natural isolation between concurent threads. To understand the cost of using Snapshot isolation, read Row Versioning Resource Usage : extra space consumed in tempdb extra space consumed in each row in the data table extra space consumed in BLOB storage for large fields
