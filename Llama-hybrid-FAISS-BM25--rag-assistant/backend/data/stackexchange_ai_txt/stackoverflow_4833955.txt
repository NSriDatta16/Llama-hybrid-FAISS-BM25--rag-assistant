The answer about specificity is on the right track. This deliberate lack of specificity is part of an approach called Object Oriented CSS (oocss), which encourages use of class selectors over ID selectors, and building in a more modular fashion, making heavy use of multiple classes on the same HTML element, and reusing classes. Also encouraged is minimizing use of !important (it breaks the cascade and lowers maintainability) and leaving redundant HTML elements out (which you are already doing to some extent, ie, using .component-heading instead of div.component-heading). Following this approach to the extent possible can greatly streamline your CSS by lowering the need for really long selectors with multiple IDs or classes. Object Oriented CSS
