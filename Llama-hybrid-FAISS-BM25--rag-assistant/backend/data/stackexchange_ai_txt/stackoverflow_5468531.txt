Unless something has changed with the new html5 stuff, no this is not valid. However, browsers usually don't care either way and javascript will be able to find these new attrs. That being said... Here's what the spec (xhtml 1.0) has to say: (from http://www.w3.org/TR/html/#uaconf ) "If a user agent encounters an attribute it does not recognize, it must ignore the entire attribute specification (i.e., the attribute and its value)."
