Is the set of metatdata that you're logging about each product likely to grow? If so, put it in another table so you can add items. If it's a concrete set that won't grow, then it probably doesn't matter. You gain a bit of conceptual elegance by using a separate table, but you give up a small amount of efficiency and complexity.
