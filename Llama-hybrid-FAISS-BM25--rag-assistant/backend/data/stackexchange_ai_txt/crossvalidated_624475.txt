Trouble Achieving Optimal Parameters for XGBoost Regressor with Sin(x) Time Series

I'm facing a challenge while attempting to optimize parameters for an XGBoost regressor (Python) using time series data. I've created a time series using the following code: import pandas as pd import numpy as np start_date = '2010-01-01' end_date = '2020-01-01' dates = pd.date_range(start=start_date, end=end_date, freq='D') unix_dates = np.array([int(pd.Timestamp(d).timestamp()) for d in dates]) freq = 'D' # Monthly frequency date_rng = pd.date_range(start=start_date, end=end_date, freq=freq) prices = np.sin(unix_dates) data = pd.DataFrame({'ds': date_rng, 'y': prices}) Now, I am attempting to train an XGBoost regressor using the following parameter combinations: 'max_depth': (3, 6, 12), 'learning_rate': (0.001, 0.01, 0.3), 'n_estimators': (1, 10, 50, 500) For parameter tuning, I've considered both MSE and MAPE. However, I'm encountering two distinct issues based on the data shuffle: Constant Output : When I don't shuffle my data, the XGBoost regressor produces a constant output. Noisy Output : Conversely, when I shuffle my data, the model generates noisy predictions. I'd greatly appreciate any insights or recommendations on how to effectively optimize the XGBoost regressor parameters for my time series data. Specifically, I'm seeking guidance on how to overcome the challenges related to the model's behavior with and without data shuffling.
