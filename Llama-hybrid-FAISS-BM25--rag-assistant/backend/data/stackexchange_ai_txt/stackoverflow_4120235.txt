You can get the necessary OS information by System#getProperty() as follows: String osName = System.getProperty("os.name"); String osVersion = System.getProperty("os.version"); String osArch = System.getProperty("os.arch"); String javaRuntimeVersion = System.getProperty("java.runtime.version"); For others, check the javadoc of System#getProperties() . The user agent string is not so relevant. It's after all the same JVM and OS. Regardless, you could let the server side programming language get it from the current request headers and pass it as an or parameter to the applet. Here's an example using JSP/EL: and obtain it in the applet as follows: String userAgent = getParameter("userAgent"); Once having an user agent string at hands, you may find the webservice at http://user-agent-string.info more useful than the user agent utils at Google Code.
