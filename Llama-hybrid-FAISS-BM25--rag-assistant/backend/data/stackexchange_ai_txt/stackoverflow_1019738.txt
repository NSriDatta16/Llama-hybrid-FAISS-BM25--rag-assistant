I suggest using curl with a range header to allow you to inspect the file type itself. curl -s -v -r0-499 -o test http://stackoverflow.com/content/img/so/logo.png * About to connect() to stackoverflow.com port 80 (#0) * Trying 69.59.196.211... connected * Connected to stackoverflow.com (69.59.196.211) port 80 (#0) > GET /content/img/so/logo.png HTTP/1.1 > Range: bytes=0-499 > User-Agent: curl/7.19.4 (i386-apple-darwin9.6.0) libcurl/7.19.4 zlib/1.2.3 > Host: stackoverflow.com > Accept: */* > Then execute: $ file test test: PNG image data, 250 x 61, 8-bit colormap, non-interlaced Now you know the mime type: image/png , the file size 3438 bytes , and the file is a 250 x 61 color PNG image.
