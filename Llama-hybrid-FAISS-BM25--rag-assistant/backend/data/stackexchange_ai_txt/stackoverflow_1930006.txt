The wiki has a goof FAQ on this topic There is generally only one instance of a given view per workbench page, but there can be several instances of the same type of editor. Editors can appear in only one region of the page, whereas views can be moved to any part of the page and minimized as fast views. Editors can be in a dirty state, meaning that their contents are unsaved and will be lost if the editor is closed without saving. Views have a local toolbar, whereas editors contribute buttons to the global toolbar. Editors can be associated with a file name or an extension, and this association can be changed by users. Some advanced use are mentioned here , about the ability to reuse an editor for several types of file. or there , for linking to an editor Views can have a different organization than editor: see fast view .
