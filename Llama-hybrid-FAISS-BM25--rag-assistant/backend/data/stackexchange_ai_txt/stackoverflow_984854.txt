According to Charles Nicholson , Visual Studio 2005 gives this error with the " do...while(0) " trick: #define MULTI_LINE_MACRO \ do { \ doSomething(); \ doSomethingElse(); \ } while(0) If you absolutely must, you can use the __pragma directive to selectively disable that warning around a particular code fragment.
