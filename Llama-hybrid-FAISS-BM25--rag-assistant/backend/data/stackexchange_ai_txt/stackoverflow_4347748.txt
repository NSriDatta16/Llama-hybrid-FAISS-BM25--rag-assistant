Winform Custom Control: Why Setter not called when used from Constructor?

I have an initial value property like this: [Category("Main")] [Description("Intial Value")] [DefaultValue(10)] public int InitialValue { get { return m_initialValue; } set { m_initialValue = value; this.TrackBar.Value = this.m_initialValue; } } So in my constructor I do this for example: this.InitialValue = 10; To my surprise when dragging the custom control on a form the setter is not called so that my trackbar value is not synchronized. Why ? Only when I change the property in dialog box the setter is called.
