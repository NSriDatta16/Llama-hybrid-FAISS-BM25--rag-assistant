Why not use file caching of the data stored in the database? You remove a potentially costly, heavily repeated set of database calls, but can retain the advantages of CMS overview of language values. Store translations in the database Have a process for generating a flat file from the database values Re-fire this process every time a translation update occurs Use the flat file in any front-end translation retrieval operations.
