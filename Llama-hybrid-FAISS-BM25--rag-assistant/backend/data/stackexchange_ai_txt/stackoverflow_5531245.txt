There were two fundamental problems with the above code, (1) RewriteRules running without conditions, (2) the co= flag for RewriteRules was failing. The following code works but is not likely the most concise solution. I did end up changing my routing strategy from directly rewriting all mobile urls to using a php file for routing. This doesn't change the core function of the rewrites and the code below is still relevant to the original question. In order to have multiple RewriteRules execute based on the same RewriteCond you can either use the chain flag, see http://httpd.apache.org/docs/current/rewrite/flags.html#flag_c , or you duplicate the RewriteCond for each desired RewriteRule. Using a RewriteRule to set a cookie can, in my experience, be very finicky. While the Mod_Rewrite RewriteRule documentation, http://httpd.apache.org/docs/current/rewrite/flags.html#flag_co , says that the only required params are name and value, I had to set all params for any success. This may be something specific to my Apache configuration but I can't determine that at this point. Also, the RewriteRule C flag appears to only works when you attempt to modify the url. For example # No substitution is attempted. The following rule fails. RewriteRule .* - [co=mobile:true:.domain.com:7200:/,NC] # A useless substitution is made. The following rule works RewriteRule (.*) $1 [co=mobile:true:.domain.com:7200:/,NC] My final code.
