It may seem like i am punting this product as this is the second time i have responded with a recommendation of this. But it seems like you need a Message Queing service, in particular a distributed message queue. ere is how it will work: Your Django App requests CMD CMD gets added to a queue CMD gets pushed to several works It is executed and results returned upstream Most of this code exists, and you dont have to go about building your own system. Have a look at Celery which was initially built with Django. http://www.celeryq.org/ http://robertpogorzelski.com/blog/2009/09/10/rabbitmq-celery-and-django/
