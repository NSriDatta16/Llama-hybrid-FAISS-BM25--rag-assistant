This Will work Like a charm as long as you don't need to access the variables that are defined inside the block. Jquery plugins are usually coded using this exact shielding construct. Use It instead of noConflict if you can because noConflict forces you to pay a lot more attention to the script ordering in your page. You will need to load prototype after jquery so that It can take ownership of the $ sign.
