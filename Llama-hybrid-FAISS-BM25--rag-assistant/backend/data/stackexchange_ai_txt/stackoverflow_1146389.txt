In addition for 1), Microsoft has built in support for embedding resource files into DLL . This will always get updated when your project is changed and recompiled. The problem is, you don't have control over caching and file name. When debugging, it's hard to pick which one to debug when everything is called "webresource.axd" something. That was hell. Would love to read how others do it, too.
