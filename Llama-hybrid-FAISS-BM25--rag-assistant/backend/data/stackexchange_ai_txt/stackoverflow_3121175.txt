I think curl --verbose/-v is the easiest. It will spit out the request headers (lines prefixed with '>') without having to write to a file: $ curl -v -I -H "Testing: Test header so you see this works" http://stackoverflow.com/ * About to connect() to stackoverflow.com port 80 (#0) * Trying 69.59.196.211... connected * Connected to stackoverflow.com (69.59.196.211) port 80 (#0) > HEAD / HTTP/1.1 > User-Agent: curl/7.16.3 (i686-pc-cygwin) libcurl/7.16.3 OpenSSL/0.9.8h zlib/1.2.3 libssh2/0.15-CVS > Host: stackoverflow.com > Accept: */* > Testing: Test header so you see this works >
