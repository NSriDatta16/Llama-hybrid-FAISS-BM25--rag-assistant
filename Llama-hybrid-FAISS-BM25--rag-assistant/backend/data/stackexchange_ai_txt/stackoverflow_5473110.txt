Use separate Integer columns. Arguments: - a integer value takes less volume in the database than its string representation - in the future you will be able to do operations search/filter/sum/average on any of those columns. - it will ensure the saved data to be integer, any attempt to save something else will be make an error to raise. They are some exceptional situation where it may be more accurate to save the data in string but we need to know more to judge that.
