The Registered Globals directive appears to be on. This feature has been DEPRECATED as of PHP 5.3.0 and REMOVED as of PHP 6.0.0. Relying on this feature is highly discouraged. Information about the register_globals ini setting As indicated by the statement above, it is highly recommended that you disable this in your php.ini file on your public server. If you don't have access to the php.ini file on your public server, the manual suggests an alternative: Please note that register_globals cannot be set at runtime (ini_set()). Although, you can use .htaccess if your host allows it as described above. An example .htaccess entry: php_flag register_globals off .
