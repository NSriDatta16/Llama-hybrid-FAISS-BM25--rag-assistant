You indeed must use Integrated Authentication when using FILESTREAM: FILESTREAM Storage in SQL Server 2008 You will need to make sure that the Windows account that the production application runs under has been added as a login in SQL Server and has been granted the same permissions as the SQL Authentication account the application currently uses. You must also ensure that the account has file system permissions to write to the FILESTREAM container.
