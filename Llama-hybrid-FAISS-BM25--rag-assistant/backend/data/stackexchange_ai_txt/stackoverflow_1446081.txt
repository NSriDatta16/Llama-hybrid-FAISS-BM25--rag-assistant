If the files are small and local, there shouldn't be much difference between XML and JSON for storage, so you should go with what's easier for you. json.org has pointers to open source JSON parsers and serializers in just about any programming language imaginable, if you decide to go that route. If eval in JavaScript is what scares you, JavaScript now has evalJSON which ameliorates the code-exploit concerns.
