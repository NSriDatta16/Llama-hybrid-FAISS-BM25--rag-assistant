If the strings are determined at compile time and never change, use a resources file. If they need persistent storage, you can also use the system registry - I am dabbling in this myself, also using resource files to reduce critical typos: // Get key RegistryKey appKey = Registry.CurrentUser.CreateSubKey(Resources.keyAppPath); // Read from key string _configStr = (string)appKey.GetValue(Resources.keyConfigStr, string.Empty); // Write to key appKey.SetValue(Resources.keyConfigStr, _configStr); Resources.keyAppPath is like "Software\[Company]\[App]" Resources.keyConfigStr is the name of the key. As others have mentioned, add a layer of encryption if security is an issue with the particular data!
