There are 3 options CREATE TABLE A ( ID INT IDENTITY(1,1) PRIMARY KEY, ... Other Columns ) CREATE TABLE B ( ID UNIQUEIDENTIFIER DEFAULT NEWID() PRIMARY KEY, ... Other Columns ) CREATE TABLE C ( ID UNIQUEIDENTIFIER DEFAULT NEWSEQUENTIALID() PRIMARY KEY, ... Other Columns ) One reason why you might prefer C rather than B would be to reduce fragmentation if you were to use the ID as the clustered index .
