I would refactor your code if: Your customer complained You found something that didn't work You found a way that the code couldn't handle a change you knew was going to happen in the future. You remembered to write unit tests that will allow you to refactor, right? *As far as the structure you have there, I've seen structures like it before. It's a little cumbersome but it is standard in many places. One thing to remember is that while its possible to dynamically add and remove columns from databases, the internal storage mechanism of the database doesn't necessarily expect you to be adding and removing these columns continuously. But I don't think this is very relevant compared to the points above, which boil down to: *Does it work?
