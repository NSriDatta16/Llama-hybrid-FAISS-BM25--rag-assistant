My team moved away from injecting primitive arguments into constructor parameters mostly because of this issue. Instead, we create an object that has properties for all of the settings that need to be injected, and then do some tricks so that the object is automatically populated by the container from the config file. You can read about it here: http://www.lostechies.com/blogs/joshuaflanagan/archive/2009/07/12/how-we-handle-application-configuration.aspx
