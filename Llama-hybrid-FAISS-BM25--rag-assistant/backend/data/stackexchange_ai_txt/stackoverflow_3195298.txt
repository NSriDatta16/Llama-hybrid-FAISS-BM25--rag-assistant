As an alternative or a starter: Those two books have a good reputation amongst Java programmers and help to make up your mind how good code should look like: Clean Code: A Handbook of Agile Software Craftsmanship (Robert C. Martin) Effective Java (2nd Edition) (Joshua Bloch) Here is another interesting web link to a book that covers code reading: Code Reading: The Open Source Perspective An interesting way to train your skills towards good code is exercising coding katas .
