XSLT extension methods must return a type that is supported within XSL transformations. The following table shows the W3C XPath types and their corresponging .NET type: W3C XPath Type | Equivalent .NET Class (Type) ------------------------------------------------------ String | System.String Boolean | System.Boolean Number | System.Double Result Tree Fragment | System.Xml.XPath.XPathNavigator Node Set | System.Xml.XPath.XPathNodeIterator The table is taken from the section Mapping Types between XSLT and .NET in this MSDN Magazine article .
