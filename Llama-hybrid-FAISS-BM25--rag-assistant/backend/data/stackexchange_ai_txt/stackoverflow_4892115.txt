Benefits of two 2-node clusters: Cheaper licensing if you're going to use Standard More storage flexibility if you may use two different SANs down the road Easier version upgrades if one cluster needs to move forward and the other needs to stay behind for legacy apps Isolated security if you have 3rd party vendor apps on there and they need to RDP in, lessens the chance that they'll screw up other services Benefits of one 4-node cluster: Easier redistribution of load (including failovers) if you're doing instance stacking
