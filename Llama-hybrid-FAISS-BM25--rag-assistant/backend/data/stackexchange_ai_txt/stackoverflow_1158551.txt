Code coverage is a measure used in software testing. It describes the degree to which the source code of a program has been tested. http://en.wikipedia.org/wiki/Code_coverage The wikipedia definition is pretty good, but in my own words code coverage tells you how much automated testing you have accounted for. 100% would mean that ever single line of code in your application is being covered by a unit test. NCover is an application for .NET
