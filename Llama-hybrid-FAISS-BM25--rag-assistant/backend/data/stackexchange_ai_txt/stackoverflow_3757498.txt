This should be a good starting point to find out how to find your SQL Agent jobs using T-SQL: View (and disable) SQL Agent Jobs with TSQL The script will list out all your jobs on your database, and when they will be run next and so forth. Using the job_name, you should also be able to find out details about your jobs using the SQL Server Agent Stored Procedures in the msdb database on your server.
