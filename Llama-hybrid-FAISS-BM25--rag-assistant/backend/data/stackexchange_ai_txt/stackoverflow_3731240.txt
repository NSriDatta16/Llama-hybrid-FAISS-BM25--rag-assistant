The following link claims that if the column is variable length, i.e. varchar then NULL takes 0 bytes (plus 1 byte is used to flag whether value is NULL or not): How does SQL Server really store NULL-s The above link, as well as the below link, claim that for fixed length columns, i.e. char(10) or int , a value of NULL occupies the length of the column (plus 1 byte to flag whether it's NULL or not): Data Type Performance Tuning Tips for Microsoft SQL Server Examples: If you set a char(10) to NULL , it occupies 10 bytes (zeroed out) An int takes 4 bytes (also zeroed out). A varchar(1 million) set to NULL takes 0 bytes (+ 2 bytes) Note: on a slight tangent, the storage size of varchar is the length of data entered + 2 bytes.
