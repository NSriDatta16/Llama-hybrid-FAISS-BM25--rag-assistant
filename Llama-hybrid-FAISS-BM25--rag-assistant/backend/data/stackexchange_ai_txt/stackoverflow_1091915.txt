With WebForms you lose pretty much everything that makes them useful without a code behind page, because then VS can't auto generate the designer file that holds the actual definitions for all your runat="server" controls. What you can do is have a common base page class, and make that generic: public class DataTemplate : Page { public T Model {get;set;} } public partial class MyCodeBehindClass : DataTemplate { ... } This would allow all the drag-drop component stuff that WebForms does to work unhindered, while also allowing you to access a strongly typed model on the page:
