There is also an end time that needs to be considered. Any job that would run after the end date/time will not be run. So it's effectively exclusive. From MSDN : SQL Server Agent automatically disables schedules that are no longer current. If you edit the schedule after it has been disabled by SQL Server Agent, you must explicitly reenable it. Schedules are disabled if: They are defined to run on a recurring schedule, and the end date has passed. There were also some problems with earlier versions of SQL Server, with regards to end dates.
