You should probably (well, definitely) make sure that your server-side code deposits its actual current time zone offset onto each page somehow. Armed with that, you will be able to figure out the difference between the client and the database server. These sorts of things are always fragile, however, so things might be better if you make sure that all time communications are made in UTC terms.
