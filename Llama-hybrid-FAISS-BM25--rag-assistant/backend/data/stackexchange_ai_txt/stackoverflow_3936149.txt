Yes, "part 3" is undefined behaviour. From the C99 spec (6.7.3, paragraph 7): An object that is accessed through a restrict-qualiﬁed pointer has a special association with that pointer. This association, deﬁned in 6.7.3.1 below, requires that all accesses to that object use, directly or indirectly, the value of that particular pointer.
