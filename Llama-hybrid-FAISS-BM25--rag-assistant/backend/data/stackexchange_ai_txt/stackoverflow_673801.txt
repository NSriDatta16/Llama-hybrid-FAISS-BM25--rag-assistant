The fastest loaded resource is always the resource that doesn't need to be loaded at all. I.e. apart from shrinking your images to reasonable sizes you should read about HTTP caching. You should instruct your web server to deliver the responses with proper caching informations so that user agents may reuse local cached versions. Mark Nottingham wrote a tutorial about HTTP caching . It's a good starting point. And this is a tutorial about apache configuration on HTTP chaching.
