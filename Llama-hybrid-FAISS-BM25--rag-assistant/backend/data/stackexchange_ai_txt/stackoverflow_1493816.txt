Don't roll your own here. Asynchronous processing routines, restartability, program faults, etc. are best managed with message queues. You could use the built-in one to Windows , go open source with ActiveMQ from the Apache Software Foundation, or leverage web services in the cloud with Amazon's Simple Queue Service . There are a lot of different ways you can go here. Investigate these options before repeating the functionality yourself. Reliability in these areas is more challenging than you might expect.
