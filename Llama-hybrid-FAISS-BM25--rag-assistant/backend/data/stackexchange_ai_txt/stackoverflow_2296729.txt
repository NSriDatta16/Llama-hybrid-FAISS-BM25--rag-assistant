Are session cookies in someway tied to IP addresses? No. As long as the browser is connecting to the same IP name serverside it doesn't matter if the clients address changes or goes via a different proxy. It's more likely that something is getting cached where it shouldn't. (assuming you are doing no validation against IP address). Have you got your webserver configured to log session ids against the request/client/user agent? (i.e. in the access log) C.
