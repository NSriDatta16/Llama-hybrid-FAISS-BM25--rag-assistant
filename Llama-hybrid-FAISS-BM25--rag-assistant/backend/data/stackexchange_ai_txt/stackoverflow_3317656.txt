Haven't tested, and don't know about readymade functionality for it, but instead of allocating new storage and moving data about, you should be able implement your own EnumeratorBinder template class, binding the two IEnumerable instances in the constructor, and implement something on the lines of IEnumerable GetEnumerable() { foreach ( X x in _enumX ) yield return x; foreach ( Y y in _enumY ) yield return y; } with suitable template constraints...
