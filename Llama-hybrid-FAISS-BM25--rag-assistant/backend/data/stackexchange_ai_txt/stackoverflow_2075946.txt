There is no THE STANDARD for doing that, it would involve a large number of fields and descriptions which may be unnecessary for a single format. So each format has its own method of Meta-Data storage But since you want to implement it in an external file (or database), You almost certainly have to implement your own schema, its not very hard. You haven't mentioned what back end you're using, SQL would be good if you're database if going to be reasonably large, and easy to search if you index the right fields. OR You can use XML / XSD combination if you like.
