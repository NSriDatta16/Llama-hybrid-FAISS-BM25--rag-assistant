I would: Break the coupling to the concrete Validation object via dependency injection: define an abstract (pure virtual) Validation class, make a concrete validation class derive from it, and pass in ("inject") a reference to the abstract Validation class in the User class's constructor. For an excellent discussion of how and why to do this in C++, see Robert Martin's 1996 article on the subject from The C++ Report . Rather than returning false or silently setting some property, raise an exception. That's what they are there for.
