The insert_at_cursor call invalidates the iterator passed in to your function. When you refer back to that iterator on the last line, GTK+ shows a warning. This behaviour is explained in the GTK+ Text Widget Overview . Fixing this is a matter of not re-using the iterator, for example: buff.insert_at_cursor(')') # This invalidates existing iterators. mark = buff.get_mark('insert') iter = buff.get_iter_at_mark(mark) # New iterator iter.backward_cursor_positions(1) buff.place_cursor(iter) (Disclaimer: I haven't used the GTK+ text widget in a long time. There's probably an easier / more elegant way to do the same thing, but this one does the job.)
