The way I learned it is that this: if (obj is Foo) { Foo f = (Foo)obj; f.doSomething(); } is slower than this: Foo f = obj as Foo; if (f != null) { f.doSomething(); } Is it slow enough to matter? Probably not, but it's such a simple thing to pay attention for, that you might as well do it.
