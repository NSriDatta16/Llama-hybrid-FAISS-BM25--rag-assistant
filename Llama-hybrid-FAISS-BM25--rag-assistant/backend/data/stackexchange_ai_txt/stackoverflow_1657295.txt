Ok while i was happy yesterday... Reality stroke back today... SAFARI reacts with not re-rendering all necessary fragments behind the moving div. Especially over tinyMCE's iframe. So i ended up with the following and this works quite well. Fades out the div -> animation to position -> Fade in only if callback is fired.. $(document).ready(function(){ $(window).scroll(function() { topParentx = $('#tinyMCEwrapper').position().top; var topTarget = "0px"; if ($(this).scrollTop() >= topParentx){ topTarget = ($(this).scrollTop()-topParentx) + "px"; $('.mceExternalToolbar').animate({opacity: 0}, 1); } $('.mceExternalToolbar').stop().animate({top: topTarget}, 200, 'swing', function(){ $('.mceExternalToolbar').animate({opacity: 1}, 100, 'swing'); }); }); });
