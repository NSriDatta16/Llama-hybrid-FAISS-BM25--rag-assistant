Maintenance is easer, integration with sharepoint is easier,database submission/retrieval is easier, and workflows are easier with infoPath. Customization is more robust, and database submission/retrieval is more robust in asp.net. Basically, use infoPath if it's easy. Use C# if it contains a lot of custom things.
