If you have design sessions before writing code or have to produce a design doc, then you could add Unit Tests as the tangible outcome of a session. This could then serve as a specification as to how your code should work. Encourage pairing on the design session, to get people talking about how something should work and what it should do in given scenarios. What are the edge cases, with explicit test cases for them so everyone knows what it's going to do if given a null argument for example. An aside but BDD also may be of interest
