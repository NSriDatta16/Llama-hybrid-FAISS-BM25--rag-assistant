I would put it all in single class and make use of sensible field/method/argument names. public class GroupCapabilities { private Map > groupCapabilities; public void addGroup(String capabilityId, Group group) { Map groups = groupCapabilities.get(capabilityId); if (groups = null) { groups = new HashMap (); groupCapabilities.put(capabilityId, group); } groups.put(group.getId(), group); } public Map getGroups(String capabilityId) { return groupCapabilities.get(capabilityId); } public Group getGroup(String capabilityId, String groupId) { Map groups = groupCapabilities.get(capabilityId); return (groups != null) ? groups.get(groupId) : null; } // Etc.. } This way the you can see at method/argument names what it expects/returns.
