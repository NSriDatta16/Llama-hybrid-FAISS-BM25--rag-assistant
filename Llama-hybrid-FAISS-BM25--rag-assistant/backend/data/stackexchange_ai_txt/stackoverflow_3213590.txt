You are organizing your hierarchical data using the adjacency list model . The fact that such recursive operations are difficult is in fact one major drawback of this model. Some DBMSes, such as SQL Server 2005, Postgres 8.4 and Oracle 11g, support recursive queries using common table expressions with the WITH keyword. As for MySQL, you may be interested in checking out the following article which describes an alternative model (the nested set model ), which makes recursive operations easier (possible): Mike Hillyer: Managing Hierarchical Data in MySQL
