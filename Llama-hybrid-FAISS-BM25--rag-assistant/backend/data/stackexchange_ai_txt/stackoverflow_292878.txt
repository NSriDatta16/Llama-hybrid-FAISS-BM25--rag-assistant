Signs of a great SQL developer

Based on their work, how do you distinguish a great SQL developer? Examples might include: Seldom uses CURSORs, and tries to refactor them away. Seldom uses temporary tables, and tries to refactor them away. Handles NULL values in OUTER JOINs with confidence. Avoids SQL extensions that are not widely implemented. Knows how to indent with elegance.
