This is one option which fits your requirements. But it's hard to tell if it is the best approach without understanding the problem domain better. # script/generate model user company_id:integer role_in_company:string class User "Company", :foreign_key => "owner_id", :dependent => :destroy belongs_to :company end # script/generate model company owner_id:integer class Company "User" has_many :employees, :class_name => "User", :dependent => :destroy end The company_id and role_in_company columns are only used for Agents and End Users. Leave them null for owners - that's what the owner_id column is for.
