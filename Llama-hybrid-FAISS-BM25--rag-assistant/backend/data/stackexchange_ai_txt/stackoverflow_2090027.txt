IMHO, this is not the best way to handle timezones in GAE. From the docs : If the datetime value has a tzinfo attribute, it will be converted to the UTC time zone for storage. Values come back from the datastore as UTC, with a tzinfo of None. An application that needs date and time values to be in a particular time zone must set tzinfo correctly when updating the value, and convert values to the timezone when accessing the value. So, To handle creation dates or last modification dates, use auto_now and auto_now_add arguments . To display dates to users, convert them like this pacific_time = utc_time.astimezone(Pacific_tzinfo()) When updating the datastore, set the tzinfo attribute using the replace function: date_from_user.replace(tzinfo=EST)
