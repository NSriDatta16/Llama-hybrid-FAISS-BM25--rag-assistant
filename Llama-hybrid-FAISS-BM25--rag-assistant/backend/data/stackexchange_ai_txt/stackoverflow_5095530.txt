Per section 3.3.4 of RFC 2965 , the user agent does not include the expiration information in the cookie header that is sent to the server. Therefore, there is no way to update an existing cookie's value while retaining the expiration date that was initially set based solely on the information associated with the cookie. So the answer to this question is: you can't do that.
