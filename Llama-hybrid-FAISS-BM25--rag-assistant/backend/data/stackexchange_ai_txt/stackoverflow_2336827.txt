Yes, deleting rows affects the index and maintenance should take place to keep the index relatively in sync with existing data. Rebuilding an index was likely unnecessary - you only need to do this if the physical fragmentation is 30 percent or more according to MS documentation. REORGANIZE is usually a better choice - think of it as defraging the index. This is a good article series on SQL Server Index Fragmentation .
