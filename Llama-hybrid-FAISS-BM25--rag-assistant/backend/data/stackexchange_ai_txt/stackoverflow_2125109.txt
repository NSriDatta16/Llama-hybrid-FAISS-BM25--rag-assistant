If the target is some version of MS SQL Server, SqlBulkCopy offers an efficient insert for large data sets this is similar to the command bcp . You can also disable/remove the non-clustered indexes before inserting, and re-create them after. In SQLite, these are usually pretty fast: .dump ?TABLE? ... Dump the database in an SQL text format .import FILE TABLE Import data from FILE into TABLE Also try: PRAGMA journal_mode = OFF FYI, you should be able to run the command line utility on Windows Mobile if you include it in your package.
