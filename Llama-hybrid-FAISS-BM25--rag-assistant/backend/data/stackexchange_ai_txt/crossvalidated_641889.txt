Modeling timeseries with strong seasonality

I have monthly national home price index data , from CoreLogic. Data is seasonally adjusted. But still has strong seasonal effect. Here is the plot for monthly changes. from statsmodels.tsa.seasonal import seasonal_decompose # Decompose the time series to find seasonality decompose_result = seasonal_decompose(national_returns, model='additive', period=12) # Plot the decomposed components fig = decompose_result.plot() fig.set_size_inches(10, 7) plt.show() We can see seasonal effect as well as volatility clustering. My plan was to fit a SARIMA(1,0,1)x(1,0,0,12) and then a GARCH model. After I fit SARIMA, I was doing additional Ljung-Box test, and the p-values are not $> 0.05$ for higher lags. from statsmodels.tsa.arima.model import ARIMA # Fit an ARIMA(1,0,0) model model = ARIMA(national_returns, order=(1,1,0), seasonal_order=(1,0,0,12)) model_fit = model.fit() # Display the model summary model_summary = model_fit.summary() # Extract the residuals residuals = model_fit.resid print(model_summary) model_fit.test_serial_correlation('ljungbox',df_adjust=True,lags=20) My questions are - How would you model this?
