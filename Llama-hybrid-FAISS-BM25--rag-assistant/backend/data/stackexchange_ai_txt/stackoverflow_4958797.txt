The main reason for the speced behavior is that we want to encourage people to add type annotations, for clarity. If in for comprehensions, they get potentially very costly filter operations instead, that's a trap we want to avoid. However, I agree that we should make it easier to specify that something is a pattern. Probably a single pair of parens should suffice. val x: String = y // type check, can fail at compile time val (x: String) = y // pattern match, can fail at run time for (x: String
