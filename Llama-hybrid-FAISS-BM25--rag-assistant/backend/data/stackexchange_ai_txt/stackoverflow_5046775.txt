From the TeamCity MSTest documentation (in the autodetection section) it looks like you do need to have VS.NET installed on the build server or at have the MSTest.exe. And again, in the TeamCity Supported Platforms and Environments documentation it clearly states: MSTest (dedicated build runner; requires appropriate Microsoft Visual Studio edition installed on build agent)
