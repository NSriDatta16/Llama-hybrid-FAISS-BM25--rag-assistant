Using a key listener (or even document listener) is inappropriate for this task. Arguably the listener model is inappropriate for input events. Do you expect your listener code to run before or after the listener code that inserts the key value? How are you going to handle copy-and-paste key strokes, and also mouse driven drag-and-drop? The way to handle this is with a DocumentFilter .
