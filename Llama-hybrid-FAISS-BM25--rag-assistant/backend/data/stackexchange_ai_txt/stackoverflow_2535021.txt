In general, relying on Excel parsing your Html is very fragile. As you have discovered, the Mac version of Excel can have troubles with it. Instead, you should consider using Spreadsheet XML or an engine like NPOI to build an actual Excel file. With either an Html document or an Xml document, Excel will throw a warning to the user on download from the web that the file is not of the same type expected (e.g., you have an Html file claiming to be an Excel file based on the content-type.). If you build an actual Excel file, you won't have the compatibility problems nor will users get this warning. Spreadsheet XML NPOI
