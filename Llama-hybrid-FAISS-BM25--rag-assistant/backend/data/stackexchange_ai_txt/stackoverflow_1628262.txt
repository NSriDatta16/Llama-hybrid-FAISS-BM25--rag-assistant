I asked myself the same question when I was working to PyTalk . If you are using Qt, you need to use QThread to be able to use the Qt framework and expecially the signal/slot system. With the signal/slot engine, you will be able to talk from a thread to another and with every part of your project. Moreover, there is not very performance question about this choice since both are a C++ bindings. Here is my experience of PyQt and thread. I encourage you to use QThread .
