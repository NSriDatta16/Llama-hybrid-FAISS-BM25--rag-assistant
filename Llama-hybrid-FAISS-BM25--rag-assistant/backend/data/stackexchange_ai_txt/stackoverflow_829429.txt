Actually, it's not about declaring the type. For output parameters with varying size, it is necessary to specify the parameter size. cm.Parameters["@OrganisationName"].Size = 50; I've read somewhere there was a bug in implementation that raises an exception whenever the size is not specified for certain data types. The whole thing makes it badly suitable for returning parameters with unknown size, like nvarchar(max). I'd recommend return values via SELECT rather that via output parameters.
