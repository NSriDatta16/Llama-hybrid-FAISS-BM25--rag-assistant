if you only have one bit column in a table, it will still take a byte to store it. The SQL Server Database Engine optimizes storage of bit columns. If there are 8 or less bit columns in a table, the columns are stored as 1 byte. If there are from 9 up to 16 bit columns, the columns are stored as 2 bytes, and so on. bit (Transact-SQL) SQL Server 2005 does the same
