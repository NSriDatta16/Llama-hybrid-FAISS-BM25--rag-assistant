No such setting/reghack exists. you would need to override the Item_Open event and change the message format from html or RTF (if either are detected) to Plaintext. This is the only way you could reliably strip out all the formatting junk without losing the text. that or write a custom parsing agent (which would seem to be a bit harder). either solution involves coding an addin to handle the open event and change the message format before displaying the message.
