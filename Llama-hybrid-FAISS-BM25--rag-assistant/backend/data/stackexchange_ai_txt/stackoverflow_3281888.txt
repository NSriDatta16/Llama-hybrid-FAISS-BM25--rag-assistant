This error could be related to permissions: Did you configure a snapshot share and give the IUSER_Machine name rights to Read and List Files and Folders? You could restart the synchronization with verbose history logging as described here: How to enable replication agents for logging to output files in SQL Server . Check these: Ensure your subscription database is not timing out on auto-grow. Make sure there is enough space to apply the snapshot on the subscriber
