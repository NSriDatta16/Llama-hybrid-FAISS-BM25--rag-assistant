AFAIK this is not possible and it will be problematic on developer's machines. File references is the way to go. I usually organize projects like this: $ -- TeamProject1 -- branches -- trunk Solution1.sln -- lib TestProject2.dll -- src -- test TestProject1.csproj references TestProject2.dll from lib -- TeamProject2 -- branches -- trunk Solution2.sln -- lib -- src -- test TestProject2.csproj This way TeamProject1 is independent from the source code of TeamProject2 and it contains all the necessary dependencies. If TeamProject2 changes it won't necessary break TeamProject1.
