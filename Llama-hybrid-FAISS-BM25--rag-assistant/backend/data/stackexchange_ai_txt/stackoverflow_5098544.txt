check The Adjacency List Model and The Nested Set Model described in Joe Celko's Trees and Hierarchies in SQL for Smarties you should find plenty information to this topic. one article is here Update The Nested Set Model is very good if you are looking for a task like 'Retrieving a Single Path' . What you have is 'Find the Immediate Subordinates of a Node' . Here the Adjacency List Model is better. | id | p_id | name | | 1 | null | root | | 2 | 1 | nd1.1 | | 3 | 2 | nd1.2 | | 4 | 1 | nd2.1 | SQL to get a row with name of a fragment and it's direct sub items. SELECT p.name, GROUP_CONCAT( c.name SEPARATOR '/' ) AS subList FROM _table p INNER JOIN _table c ON p.id = c.p_id WHERE p.name = 'root' P.S. prefer WHERE p.id = 1 . Id is unique where as name can be ambiguous. see MySQL GROUP CONCAT function for more syntax details.
