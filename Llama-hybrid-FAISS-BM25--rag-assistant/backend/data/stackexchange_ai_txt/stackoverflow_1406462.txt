You cannot guarantee that the automatically-generated WSDL will match the WSDL from which you create the service interface. In your scenario, you should place the WSDL file on your web site somewhere, and have consumers use that URL. You should disable the Documentation protocol in the web.config so that "?wsdl" does not return a WSDL. See Element . Also, note the first paragraph of that article: This topic is specific to a legacy technology. XML Web services and XML Web service clients should now be created using Windows Communication Foundation (WCF) .
