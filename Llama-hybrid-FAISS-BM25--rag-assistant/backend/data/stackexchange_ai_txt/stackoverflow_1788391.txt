Given that you already have the data in a database, you could send the user an encrypted SQLite database. There are various (paid) third party tools to encrypt data in SQLite transparently, or else you can just write encrypted data into SQLite in blobs, depending on the actual usage. Also, depending on the technologies involved, you might be able to leverage the Microsoft Crypto API in SQLite ADO.NET driver, for instance.
