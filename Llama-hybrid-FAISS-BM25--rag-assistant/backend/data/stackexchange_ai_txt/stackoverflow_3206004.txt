Alternatively to JS Bangs' answer , you can have GCC treat them as system headers , which disables all warnings (excepting #warning directives) for those headers. If the -isystem switch is unhelpful, you can wrap all of the offending headers with simpler headers that contain only the appropriate line: #pragma GCC system_header
