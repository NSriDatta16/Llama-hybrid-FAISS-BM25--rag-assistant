First, you're not running it many times and comparing averages. Second, your method isn't being jitted until it actually runs. So the just in time compile time is added into its execution time. A true test would run each multiple times and average the results (any number of things could cause one or the other to be slower for run X out of a total of Y), and your assemblies should be pre-jitted using ngen.exe .
