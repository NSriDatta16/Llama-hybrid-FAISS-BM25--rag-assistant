Isolated Storage is, in effect, a location on the local file system that is determined using the user information and the assembly information so if you put files there then they should be fairly free from corruption due to other apps selecting the same location. It gets around the issue of everyone deciding to use c:\todo to store to-do lists for example. There's a pretty good MSDN article that's worth a read.
