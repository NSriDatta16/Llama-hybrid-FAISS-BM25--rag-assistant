i would indeed go for (a.o.) localstorage. I wrote a small proof of concept of such an offline web-app earlier this year (cfr. this blogpost and the offline-enabled webapp here ), the basic approach being; put data in arrays/ objects use standard javascript functions to do CRUD (or go for jlinq ) json-ify the array/object for storage use a storage-abstraction library like persistjs to store/ retrieve json-ified array/object
