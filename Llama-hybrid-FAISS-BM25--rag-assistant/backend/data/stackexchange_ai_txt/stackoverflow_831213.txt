Yes. You have to wrap the .dll in a .jar file, and reference it in your .jnlp file using See here for a more detailed description. If you want to run an executable, then your best bet is to package it up in your .jar as a resource, explode it to (say) a temporary directory, and then run it using the normal Process/Runtime mechanism.
