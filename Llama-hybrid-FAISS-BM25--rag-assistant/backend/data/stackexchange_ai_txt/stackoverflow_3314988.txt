VARBINARY(MAX). If you're using SQL Server 2008, then FILESTREAM is also an option that should be considered. According to Microsoft's guidelines, consider FILESTREAM when: Objects that are being stored are, on average, larger than 1 MB. Fast read access is important. You are developing applications that use a middle tier for application logic.
