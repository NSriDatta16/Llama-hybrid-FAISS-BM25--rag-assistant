In most organizations, you'll have a set of Cognos environments, for example : Development (this is where BI content authors create models, reports, etc) QA (this is were testers validate that the BI content is accurate) Production (where real end users consume the BI content) A BI Lifecycle Management tool like MotioCI can make promoting content between multiple Cognos environments far easier (and repeatable / traceable). For example, here's a short video which demonstrates the process : http://www.motio.com/products/ci/video/labelingAndPromotingBiContent.do
