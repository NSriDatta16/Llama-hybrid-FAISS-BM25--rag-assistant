It's best to think of your usage scenarios when planning out your storage layer. This will help you figure out which queries you'll want your persistence layer to optimize for. If you're going to handle lot's of queries like, "Find me all objects within this space". You may want to look at using the spatial extensions. However, if most of the time you're simply drawing an object with a given id, simply saving the polygons as a json blob in the DB may do. CREATE TABLE Polygons ( polygon_id int not null, vertices_json varchar(4096) )
