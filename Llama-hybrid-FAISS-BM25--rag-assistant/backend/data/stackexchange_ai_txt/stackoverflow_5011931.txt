You cannot use InProc in a multithreaded way (3 threads in the app pool) without guaranteeing that each request goes to the same thread. InProc literally means in process, as in the memory of the current thread. If you move to another storage method, say, to the database, then session can be shared across all instances of a given application. I suggest reading this
