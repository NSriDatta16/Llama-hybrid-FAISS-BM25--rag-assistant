No, there is no such type. But there are some choices: Dynamically generated queries (sp_executesql) Temporary tables Table-type variables (closest thing that there is to a list) Create an XML string and then convert it to a table with the XML functions (really awkward and roundabout, unless you have an XML to start with) None of these are really elegant, but that's the best there is.
