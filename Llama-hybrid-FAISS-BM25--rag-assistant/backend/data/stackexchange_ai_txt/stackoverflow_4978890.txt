The InitDataProxy could extend ValueProxy instead, which doesn't require that the object on the server have any kind of id or version semantics. The domain-side InitData type could be an interface, possibly implemented with an anonymous type. interface InitData { User getUser(); Organization getOrgatization(); } class InitService { static InitData makeInitData() { return new InitData() { ..... }; } } @ProxyFor(InitData.class) interface InitDataProxy extends ValueProxy { UserProxy getUser(); OrganizationProxy getOrganization(); } @Service(InitService.class) interface Init extends RequestContext { Request makeInitData(); }
