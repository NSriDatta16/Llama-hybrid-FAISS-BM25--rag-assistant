Assuming that you have followed best-practice and put your internal classes in different packages to your public APIs, you can run javadoc with the public API package names as command line arguments. Refer to the javadoc command line synopsis for more details. (If you haven't organized your packages to keep internal classes out of API packages, you may be in for a bit of pain ...)
