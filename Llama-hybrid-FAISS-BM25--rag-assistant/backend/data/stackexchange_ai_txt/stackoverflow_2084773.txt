It's called refactoring buddy and it's good, you just need to limit it so you won't end up wasting all your time refactoring code you have and is working instead of writing new code. Some of the reasons why one must refactor are: Enhancing performance. Organizing code. You need to write your code in a different way to get something to work. To do something in a different way because it saves a lot of work (i.e.: Using MXML instead of ActionScript). You used the wrong name for a variable.
