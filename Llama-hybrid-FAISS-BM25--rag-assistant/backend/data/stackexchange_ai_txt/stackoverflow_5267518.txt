From the omp_set_dynamic(n) maual page: n ... A value that indicates if the number of threads available in subsequent parallel region can be adjusted by the runtime. If nonzero, the runtime can adjust the number of threads, if zero, the runtime will not dynamically adjust the number of threads Try to set the number of threads you want with a #pragma omp parallel num_threads(n) turning of the omp_set_dynamyc.
