I recommend using Agent instead: Create Agent proxy account for the account that will run the job Create Agent job that runs this package Make it use the proxy account created in #1 Test the job In SQL code, use sp_start_job to start this job The downside is that you can't easily pass the parameters from SQL to the package this way.
