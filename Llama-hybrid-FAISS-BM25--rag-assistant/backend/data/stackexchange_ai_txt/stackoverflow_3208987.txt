Scheme-relative URL's are particularly useful when you're serving a HTTPS website and you would like to share the same static content like stylesheets, images, scripts and so on as from a HTTP site. Hardcoding http: in the static content links like , , and so on and viewing the webpage itself over https: would cause the average webbrowser to pop a security warning like the following well-known IE security alerts: Serving the "nonsecure" content over a scheme-relative URL would fix this issue.
