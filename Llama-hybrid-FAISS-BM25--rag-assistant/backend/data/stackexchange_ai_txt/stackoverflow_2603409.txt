If you can reorganise the code to do this, build the shared .obj files into a library project, then you can link with that library from the dependent projects. That's the only "clean" way I can think of to solve the problem - anything else I can think of is likely to involve "fighting" or "tricking" visual studio, which is usually a bad idea.
