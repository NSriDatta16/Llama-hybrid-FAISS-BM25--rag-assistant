Given the hard-to-dispute fact that = { 0 } is infinitely more readable than memset(..., ..., ... sizeof ...) , then the following would discourage explicitly using memset : In Visual Studio 2005, compiling for Windows Mobile, full optimized release build: ; DWORD a[10] = { 0 }; mov r3, #0 mov r2, #0x24 mov r1, #0 add r0, sp, #4 str r3, [sp] bl memset add r4, sp, #0 mov r5, #0xA ; DWORD b[10]; ; memset(b, 0, sizeof(b)); mov r2, #0x28 mov r1, #0 add r0, sp, #0x28 bl memset add r4, sp, #0x28 mov r5, #0xA Pretty much the same.
