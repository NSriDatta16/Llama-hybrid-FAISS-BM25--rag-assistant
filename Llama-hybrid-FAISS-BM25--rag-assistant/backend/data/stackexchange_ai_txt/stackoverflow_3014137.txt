can/should iCal property parametrs be escaped?

specifically, the CN (common name) parameter, e.g. ORGANIZER;CN=John Doe,Eng:mailto:jd@some.com The RFC is vague on this, IMHO. It is very clear about property values of type TEXT, but for this parameter it just says " The parameter value is text ". Anyways, the escaping specified for TEXT type doesn't seem complete for parameter values (e.g. ':' is not escaped). thanks a lot!
