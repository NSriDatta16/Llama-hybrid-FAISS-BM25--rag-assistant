No, a SELECT that is already executing that the moment of the INSERT will never gather new records that did not exist when the SELECT statement started executing. Also if you use the transactional storage engine InnoDB, you can be assured that your SELECT will not include rows that are currently being inserted. That's the purpose of transaction isolation , or the "I" in ACID . For more details see http://dev.mysql.com/doc/refman/5.1/en/set-transaction.html because there are some nuances about read-committed and read-uncommitted transaction isolation modes.
