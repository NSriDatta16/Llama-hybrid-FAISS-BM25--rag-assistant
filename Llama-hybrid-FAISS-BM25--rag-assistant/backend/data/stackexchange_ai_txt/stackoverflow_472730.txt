Tomcat can typically be monitored via JMX; as can be seen from this documentation . This will normally be the cleanest way to monitor, since this can be integrated into a number of third-party monitoring tools that can support all kinds of notifications and bells and whistles. This may still not give you the full "outside view" picture if your site is up and visible to customers, since there is a number of other things that can go wrong. Quite a few of the management systems also allow you to install agents that do this and integrate into the total monitoring scheme. There are a number of open source jmx tools
