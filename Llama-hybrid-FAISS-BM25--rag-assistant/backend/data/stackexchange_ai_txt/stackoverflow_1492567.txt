This is a very common question, people ask about throwing exception from WCF to Silverlight. The answer depends on the version of Silverlight you use. Prior to Silverlight 3, catching exceptions/faults was not supported. The most elegant and generic solution for SL2 is described here: http://www.codeproject.com/KB/silverlight/SilverlightExceptions.aspx Since Silverlight 3 was released, there is a partial support, which can be used with some tricks, as described here: http://www.netfxharmonics.com/2009/07/Understanding-WCF-Faults-in-Silverlight-3
