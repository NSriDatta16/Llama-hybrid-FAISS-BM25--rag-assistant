This is because you've got them in a Folder in the solution. The pragmatic solution is to simply add them to the solution root, the actual location of the file doesn't matter. That perhaps creates a wee bit of clutter in your Solution Explorer window. If that's unacceptable then a pre-build event that uses xcopy /d is the workaround. xcopy /d "$(ProjectDir)DLLs\*.dll" "$(TargetDir)"
