GCC is quite smart. Consider this code fragment: #include void __inline__ inc(int *val) { ++ *val; } int main() { int val; scanf("%d", &val); inc(&val); printf("%d\n", val); return 0; } After a gcc -S -O3 test.c you'll get the following relevant asm: ... call __isoc99_scanf movl 12(%rsp), %esi movl $.LC1, %edi xorl %eax, %eax addl $1, %esi movl %esi, 12(%rsp) call printf ... As you can see, there's no need to be an asm expert to see the inc() call has been converted to an increment instruction.
