You may create static wrapper class for kernel. That way you could do something like ServiceLocator.Resolve() For registering services there are two ways: inline and module registration. Both of them should be loaded at bootstrapping. Module is better for organizing. Maybe it would be easier to start with StructureMap because there is static class and it has auto mapping features. Those screencasts should get you starting: Dime Casts series for Ninject Dime Casts series for StructureMap
