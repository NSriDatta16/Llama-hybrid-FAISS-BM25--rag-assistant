On smartphone browser, things are a bit complex as different versions of browser behaves differently to javascript/css. For example android browser doesn't support html5 till android os 2.0 and in later os versions it does support html5 video tag but not audio tag. Similarly there are variations on mobile safari. So the best way is looking at user-agent string and parse it for os versions/platform / os and act accordingly.I wrote a small function to retrieve some info from browser user-agent. var agentInfo=(function(){ var info={}, ua=navigator.userAgent.split(/[()]/)[1].split(/;/); info.platform=ua[0] && ua[0].toLowerCase(); info.version=ua[2] && ua[2].match(/[0-9._]+/g)[0]; info.os=ua[2] && (temp=ua[2].match(/[^0-9_.]+/g)[0].toLowerCase()) && (temp=/(windows)|(linux)|(mac)|(iphone)|(android)/.exec(temp)) && temp[0]; return info; })(); Now based on os/platform/version you can write appropriate css.
