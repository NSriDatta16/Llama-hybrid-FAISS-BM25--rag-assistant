You can generate thin, screen-oriented polygons instead, and set the fragment's alpha according to the distance to the line. Example : a (0,1) b (0,1) +--------------------------------------+ A | | B ----+--------------------------------------+---- | | +--------------------------------------+ d (0,0) c (0,0) Suppose you want to draw segment [AB]. Draw polygon abcd instead Map the UVs (the (0,0) , (0,1)) bind a 8x1 black and white texture that is white only on the center render with a fragment shader that set gl_FragColor.a from the texture (more or less the technique used in ShaderX5) But do this only if you can't use MSAA.
