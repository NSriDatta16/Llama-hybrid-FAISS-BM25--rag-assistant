I got a different error message when running your script under /bin/sh , but when I switched to /bin/bash , it worked fine: $ cat foo.sh #!/bin/sh if [[ $# -lt 1 ]]; then echo "ERROR Environment argument missing" RC=50 exit $RC fi $ sh foo.sh foo.sh: 6: [[: not found $ bash foo.sh ERROR Environment argument missing You've built in a bashism . This may or may not be a big deal for your organization. If you want to keep using bash -specific features, change the shebang line to #!/bin/bash and see if that helps.
