Generally there are two widely used ways to name fields (always using camelCase ): Using an underscore prefix void F(String someValue) { _someValue = someValue; } Using this. to access the field and avoid name conflicts void F(String someValue) { this.someValue = someValue; } Personally I prefer the later, but I will use whatever convention is set forth by the organization I work for.
