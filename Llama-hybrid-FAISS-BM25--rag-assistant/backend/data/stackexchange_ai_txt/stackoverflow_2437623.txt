I would recommend against the latter style, since it mixes data retrieval with "busines logic" (i.e. what to do with the data). With the first approach you keep a high degree of freedom in what you want to do with the data, e.g. public static void main(String[] args) { System.out.println(getAgent().getAgentName()); System.err.println(getAgent().getAgentName()); someTextArea.append("The agent name is: " + getAgent().getAgentName()); // etc } If you chose to do things this way at least name the functions with names that clearly describes what is done, e.g. private static void getAgentNameAndPrintIt(Agent agent) { System.out.println(agent.getName()); }
