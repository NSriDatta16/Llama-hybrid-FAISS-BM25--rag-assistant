Yes, IE6 caches locally, provided the HTTP headers are set correctly. What do your HTTP headers look like? You can see them with a web debugger such as Fiddler. There are some known bugs in IE6 that relate to the way it caches HTC files (such as the one referenced in the code fragment you posted): they can get loaded twice, even when caching is enabled.
