As per W3C Recommendation it was mainly useful to hide the script data from USER AGENTS. Quoted from the W3c page : Commenting scripts in JavaScript The JavaScript engine allows the string " to occur at the start of a SCRIPT element, and ignores further characters until the end of the line. JavaScript interprets "//" as starting a comment extending to the end of the current line. This is needed to hide the string "-->" from the JavaScript parser.
