One problem with non-mainstream languages (such as HAML) is that tools are hard to find, because they are hard to build. This technical paper Branch Coverage for Arbitrary Languages Made Easy (I'm the author) describes how to build test coverage tools for langauges in systematic way to help get around this problem, using a generic tool-building infrastructure.
