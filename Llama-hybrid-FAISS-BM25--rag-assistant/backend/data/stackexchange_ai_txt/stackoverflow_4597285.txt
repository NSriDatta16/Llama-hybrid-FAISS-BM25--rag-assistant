It depends on where the strings will be used. If "stop" will never be used anywhere but in a menu, calling it "menu_stop" is a good idea. If it'll be used all over the place then it should just be called "stop". Also, XML comments are very useful for organizing resources. GameController Stop Start Preferences Back Finally, if you find you have tons and tons of string resources you may want to go so far as to separate them into different xml files: menu_strings.xml, dialog_strings.xml, etc. menu_strings.xml Stop Start Preferences Back dialog_strings.xml Yes, cancel. No, do not cancel.
