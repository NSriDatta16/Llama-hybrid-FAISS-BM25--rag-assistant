Cases one and two are actually the same scenario. You should set Cache-Control: public and then generate a URL with includes the build number / version of the site so that you have immutable resources that could potentially last forever. You also want to set the Expires header a year or more in the future so that the client will not need to issue a freshness check. For case 3, you could all of the following for maximum flexibility: "Cache-Control", "no-cache, must-revalidate" "Expires", 0 "Pragma", "no-cache"
