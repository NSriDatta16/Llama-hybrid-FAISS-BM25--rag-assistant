The order of the cababilities is important, see this doc www.twain.org/docs/CapOrderForWeb . EDIT: These are some code fragments from a solution Setup the auto feed capFeederEnabled = _twEntities.GetCapability(TwCap.FeederEnabled, (short)1); TwRC rc = DScap(_applicationId, _sourceId, TwDG.Control, TwDAT.Capability, TwMSG.Set, capFeederEnabled); TwCapability cap = _twEntities.GetCapability(TwCap.XferCount, 1); rc = DScap(_applicationId, _sourceId, TwDG.Control, TwDAT.Capability, TwMSG.Set, cap); and then when the when the Twain Window Message is sent rc = DSixfer(_applicationId, _sourceId, TwDG.Image, TwDAT.ImageNativeXfer, TwMSG.Get, ref hbitmap); rc = DSpxfer(_applicationId, _sourceId, TwDG.Control, TwDAT.PendingXfers, TwMSG.EndXfer, pxfr); finally reset the scanner for the next document rc = DSpxfer(_applicationId, _sourceId, TwDG.Control, TwDAT.PendingXfers, TwMSG.Reset, pxfr);
