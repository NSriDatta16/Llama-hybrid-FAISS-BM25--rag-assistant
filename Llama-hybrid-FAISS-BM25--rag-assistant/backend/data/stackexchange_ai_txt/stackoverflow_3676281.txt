1). IMHO you will have bulky hierarchy. It's better, in my opinion, to have adapters for group of controls. Infragistics, for instance, have complicates and, in many ways, illogical inheritance hierarchy. If I had to implement skinning quickly, I'd implemented skinning logic in a common form (or other container) class instead of creating large set of own controls.
