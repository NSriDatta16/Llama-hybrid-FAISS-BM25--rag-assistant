Certainly the best approach is to leverage the Search API (or whatever relevant API) from Twitter to accomplish this. But if, for some other project, you want to host a full browser environment (JS + dom + renderer) that remains alive and lets you interact with it, the PhantomJS project might work: http://code.google.com/p/phantomjs/ It hosts a WebKit instance for automation.
