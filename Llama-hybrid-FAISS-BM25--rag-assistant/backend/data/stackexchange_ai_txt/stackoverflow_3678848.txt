A lot of people claim HTML5 will kill off plugins like Flash and Silverlight, but that simply isn't the case. Take for instance, this scenario. Load a binary file over a url. For the sake of the demo, we'll use shapefile format (.shp), which contains geographical polygons in a coordinate system. Parse the byte contents client side. Draw the contents in an efficient manner. Demo: http://www.libertyvanguard.com/liberty-gis This is just one example where plugin technology won't be replace by HTML5.
