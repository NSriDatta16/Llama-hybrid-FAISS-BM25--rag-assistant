Regarding Question 1 - yes, you need to do both. You would expose your applications scripting interface to an embedded interpreter, which then runs the user script in question. The Python manuals section on embedding includes a section Extending Embedded Python and there are tutorials for similar things for Lua, see e.g. this article . Of course there are probably easier ways to be found when you have decided with what scripting language and binding mechanism you want to go.
