The separation of application in persistence lair and service lair is needed to help you as a developer to maintain some structure that is easy to understand and maintain, it's developers responsibility to follow some pattern and use code properly. It's not framework which restricts you (developer) from using it's incorrectly, DDD is design pattern (a guidance how to organize you application structure).
