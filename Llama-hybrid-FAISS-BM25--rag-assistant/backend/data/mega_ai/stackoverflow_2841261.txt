Trying to parse, or validate, HTML is a complex job best not attempted with a regex. There are just too many possibilities for it to be efficient. Jakub got there before me, but I agree. Use tools that exist for the job like HTML Tidy - http://tidy.sourceforge.net/ It can fix invalid HTML, see a nice overview at http://www.w3.org/People/Raggett/tidy/ There is some PHP integration at https://www.php.net/tidy
