No . OO and imperative are two orthogonal concepts. For example: The Common Lisp Object System is an example of Lisp OO and maybe the most complex object system around. OCaml is a functional language with an object system and a module system supporting object-oriented organization Scala is a functional language with a very flexible OO system Haskell allows you to write object-oriented code using higher-kinded polymorphism There are lots of different ways to be object-oriented.
