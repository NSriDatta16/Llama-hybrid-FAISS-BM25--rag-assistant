Field.Store is explained beautifully in this SO thread Lucene indexing: Store and indexing modes explained Basically the search hits will include the data for all the fields with Field.Store.YES set, you don't need this if you have another storage mechanism like a DB. If you do rely on Lucene for this exclusively, it makes sens to store a few common fields, at least one that allows you to get to the original document on disk.
