This is caused if you have a custom migration policy that returns NO without setting the error pointer. e.g. - (BOOL)createDestinationInstancesForSourceInstance:(NSManagedObject *)sInstance entityMapping:(NSEntityMapping *)mapping manager:(SVMigrationManager *)manager error:(NSError **)error; { return NO; } Check the code carefully to find any situations where this could occur.
