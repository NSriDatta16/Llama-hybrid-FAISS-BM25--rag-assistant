Namespaces In larger JavaScript applications or frameworks it can be useful to organize the code in namespaces. JavaScript doesn't have a module or namespace concept buildin but it is easy to emulate using JavaScript objects. This would create a namespace called ns and attaches the function foo to it. if (!window.ns) { window.ns = {}; } window.ns.foo = function() {}; It is common to use the same global namespace prefix throughout a project and use sub namespaces for each JavaScript file. The name of the sub namespace often matches the file's name. The header of a file called ns/button.js could look like this: if (!window.ns) { window.ns = {}; } if (!window.ns.button) { window.ns.button = {}; } // attach methods to the ns.button namespace window.ns.button.create = function() {};
