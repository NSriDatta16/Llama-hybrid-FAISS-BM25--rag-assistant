The "default constructors" of primitive types (including pointers) produce 0-filled memory, much like global variables. Here is the relevant standard language (from dcl.init): To default-initialize an object of type T means: --if T is a non-POD class type ( class ), the default constructor for T is called (and the initialization is ill-formed if T has no acces- sible default constructor); --if T is an array type, each element is default-initialized; --otherwise, the storage for the object is zero-initialized. ... 7 An object whose initializer is an empty set of parentheses, i.e., (), shall be default-initialized. Also, from lib.map.access: 23.3.1.2 map element access [lib.map.access] reference operator[](const key_type& x); Returns: (*((insert(make_pair(x, T()))).first)).second.
