Personally I wouldn't recommend making code regions part of your code convention. The main reason being that regions hide code , which could potentially lead to issues like: Developers might miss some important part of the source code The average amount of LOC in the same file tends to increase If you are interested in enforcing a coding style convention in your team, have a look at Microsoft StyleCop . Note that the tool currently only works for C#.
