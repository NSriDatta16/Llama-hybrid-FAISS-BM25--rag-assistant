According to your jQuery question : Each jQ object has a data property. It does not stored inside the element itself - it's very important. jQ use general storage for all elements - jQuery.cache . So when you add anything to the element like this: $('#myEl').data('someValue', 1); jQ do the following: jQuery.cache[elementUniqId]['someValue'] = 1; So element does not contain its data object. It only have an uniq id that is allows it to access to the data recorde at the global storage. ( elementUniqId is autogenerated) jQ events are stored into the element data as well: $('#myEl').click(function() { first listener }); $('#myEl').mouseenter(function() { one more listener }); $('#myEl').click(function() { anotheer listener }); Will be stored: jQuery.cache[elementUniqId]['events'] = { click: [function() { first listener }, function() { anotheer listene }], mouseenter: [function() { one more listener }] }; It allows jQ to store the order of execution for all listeners attached to each event. And later, when you delete dom element, using jQuery - .remove() , jQuery loops through the jQuery.cache[elementUniqId]['events'] and remove each listener from the element, and after removes element cache record. It allows jQ to preven memory leaks
