You certainly need to include the specific jdbc driver your application need. And you can have several jars in the jnlp resources. You might need to specify security constraints too, jnlp applications runs in a sandboxed environment by default, so it might not be allowed to connect to another host (like your database server). But show the SQL exception, it should contain clues th what's wrong. Here is one nice place to start
