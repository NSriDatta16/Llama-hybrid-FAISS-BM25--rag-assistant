To do so I think you'd basically have to have a Tomcat (or equivalent) server running and, while you're constructing your email, send a request to the server to invoke your JSP, which would then send back the contents of the email (or the HTML part of the email) as the response. It doesn't seem very elegant, though. You'd probably be better off with a separate templating system. Here's an interesting-looking answer from a related question.
