The QueryContinueDrag event is raised on the drag source . Checking for the state of the keys you are interested in is going to require pinvoke, the event is only designed to help recognize the Escape key and modifier key state changes. Which is something to keep in mind, that these keys have any special action is very undiscoverable. [DllImport("user32.dll")] private static extern short GetKeyState(Keys key); It returns a value
