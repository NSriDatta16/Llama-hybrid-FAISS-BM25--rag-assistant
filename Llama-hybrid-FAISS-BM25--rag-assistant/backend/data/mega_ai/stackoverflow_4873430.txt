There are two main approachs to this: You can move the application in house, without making any changes by running on Azure Appliance. http://www.microsoft.com/windowsazure/appliance/ You can architect the solution so that you do not use functionality that is specific to Azure. For example store data in Azure SQL not in Azure Table Storage. A web frontend built in ASP.Net MVC will run in Azure and on IIS.
