The following reasons apply: Separating your application into templates with an engine makes your application less vulnerable to halting code errors Using templates can give you greater flexibility in the future when refactoring because the namespace won't be directly built into the application Using templates encourages (forces) developers to keep business logic and code OUT of the presentation layer. Using templates it is easier to mock up datasets and pass them to a template engine and get a preview of what the site will look like with data
