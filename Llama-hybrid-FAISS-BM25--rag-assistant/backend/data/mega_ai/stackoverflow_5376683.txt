I personally found this to be the correct and elegant way to determine if the date is both according to format and valid : treats dates like 20111-03-21 as invalid - unlike checkdate() no possible PHP warnings (if any one parameter is provided, naturally) - unlike most explode() solutions takes leap years into account unlike regex-only solutions fully compatible with the mysql DATE format (10.03.21 is the same as 2010-03-21) Here's the method you can use: function isValidMysqlDate( string $date ): bool { return preg_match( '#^(?P \d{2}|\d{4})([- /.])(?P \d{1,2})\2(?P \d{1,2})$#', $date, $matches ) && checkdate($matches['month'],$matches['day'], $matches['year']); }
