The fundamental issue for a DLL in kernel mode is whether the DLL calls any user-mode code . If a DLL contains anything other than native kernel API calls , you'll get linker errors if you try to link your driver with it when you build (and the kernel wouldn't load it anyway) check the following link Calling a DLL in a Kernel-Mode Driver Edit: Another useful link DLLs in Kernel Mode Tim Roberts
