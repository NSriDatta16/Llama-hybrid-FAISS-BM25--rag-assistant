Standard list, maybe not, but you have some examples : syntax: regexp \.*py[co] \.DS_Store ~$ \.coverage \.egg-info syntax: glob nbproject app.yaml auth.py dist target WEB-INF/appengine-generated Basically, at least any directory with generated content should be ignored. The same principles holds true for Java app projects like this one or that one : syntax: glob *~ *.patch *.sedbak */target/* */ searchindex/* */test-output/* hs_err_pid*.log tomcat syntax: regexp \.jar$ ^\.pc/ ^.ant-targets-build.xml \.pages.xml.spdia$ temp-testng-customsuite.xml$ # eclipse and maven stuff ^target # kde related \.directory$ #gwt related ^ -war/war/WEB-INF/classes/ ^ -war/tomcat \.gwt-tmp$ ^ -war/org.fedorahosted. .webtrans.Application ^ -war/war/org.fedorahosted. .webtrans.Application Off course, I will keep any Eclipse or maven related file under source control , in order to facilitate the build step when anyone will clone the repo.
