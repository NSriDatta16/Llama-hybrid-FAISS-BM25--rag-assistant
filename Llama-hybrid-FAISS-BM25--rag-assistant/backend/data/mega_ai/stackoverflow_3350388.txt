In Python, you should consider using the multiprocessing module instead of threads, especially if you have a multicore machine: multiprocessing is a package that supports spawning processes using an API similar to the threading module. The multiprocessing package offers both local and remote concurrency, effectively side-stepping the Global Interpreter Lock by using subprocesses instead of threads. Due to this, the multiprocessing module allows the programmer to fully leverage multiple processors on a given machine. It runs on both Unix and Windows Also consult the following for examples and an introduction. multiprocessing Basics Communication between processes with multiprocessing
