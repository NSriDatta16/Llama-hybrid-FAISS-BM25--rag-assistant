I would not advise using a librarian to take Windows' library contents into your own library -- it's likely that that's against the license. I see two possibilities Documenting the dependency Using a #pragma in your .h file that requests the .lib to be linked against. If VS can find it, it's the same as including it on your link line. http://msdn.microsoft.com/en-us/library/7f0aews7(VS.80).aspx #pragma comment(lib, "libname.lib")
