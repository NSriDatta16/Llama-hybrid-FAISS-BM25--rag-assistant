You might want to check a weird side effect of abort() When the abort() method is used, the readystatechange event fires in Explorer and Mozilla. Worse, readyState = 4, which means that the average xmlhttp script assumes the data has been loaded correctly. This can give very weird effects. documented here: http://www.quirksmode.org/blog/archives/2005/09/xmlhttp_notes_a_1.html
