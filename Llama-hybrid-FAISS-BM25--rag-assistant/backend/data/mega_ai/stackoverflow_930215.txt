No, I don't think they're the same. The centralized (e.g., SVN, CVS, etc.) versus distributed (Git, Mercurial, etc.) repository is a fundamental difference that reflects the way your development team operates. I use SVN. I think that my organization's collective heads would explode if a distributed SCM system like Git was introduced. It's such a different concept for most people.
