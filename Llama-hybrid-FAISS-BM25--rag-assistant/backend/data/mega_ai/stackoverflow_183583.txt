I think a framework that strives to be "rails-like" refers to several things: model-level: an AR-patterned ORM (rather than datamapper), migrations or some automated schema and model-layer management, handling foreign keys in the application (not in database schema, and also not using stored procedures or pure DBMS logic) TDD encouraged: automatically generated skeletons for unit-tests, naming conventions connecting database table names and model names, controller and view actions and HTML templates streamlined route recognition and route generation scheme emphasis on REST architecture integration with ajax libs: RJS, prototype and scriptaculous
