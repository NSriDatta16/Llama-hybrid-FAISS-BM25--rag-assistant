You don't actually have to create a second solution, or pull out OS-agnostic code. There are a few tricks to it though. Generally: Reference the MonoTouch assemblies, even in your .Net 3.5/Mono Test Projects A Supervisor Controller pattern helps with the amount of code you can test. Have written up a few details on what we have found to be best practice so far. You can find it here: http://ben.phegan.name/index.php/2011/02/28/monotouch-and-unit-testing .
