Client side cookies have best performance, No round trip to web server is a big win for performance. But Cookie has size limitation. see following link about limitation on IE, Other browser should have similar limitation. http://support.microsoft.com/kb/306070 , cookies are used for small amount day storage, like session key. Session normally means one of server process, if you use on a web farm, Session can not be shared across multiple web server. If you have a single web server, session should be best way to store information on the server side. For database, it is most flexible solution, but it has performance hit. for high performance website, proper caching is key to go.
