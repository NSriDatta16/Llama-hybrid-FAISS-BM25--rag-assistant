Here's what the insides of our SetThreadName looks like (in a mixed native C++, C++/CLI, C# app): #pragma warning(disable: 6312 6322) __try { RaiseException( 0x406D1388, 0, sizeof(info) / sizeof(DWORD), (DWORD*)&info ); } // don't implement MSDN's suggested fix for 6312 here - it causes a nasty unhandled exception // to bubble up into managed code. Disable the compiler warning instead. __except(EXCEPTION_CONTINUE_EXECUTION) { } #pragma warning(default: 6312 6322)
