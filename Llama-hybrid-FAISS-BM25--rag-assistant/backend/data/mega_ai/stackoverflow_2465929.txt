The biggest value of a unit test is for maintenance, to ensure that the code still works after changes. So, concentrate on methods/classes that are most likely / most frequently changed. Next in importance are classes/methods with less-than-obvious logic. The unit tests will make them less fragile while serving as extra "documentation" for their contracted API
