It is illogical to reinvent anchor behaviour. It is better to not expose hash links to insignificant fragments (although modern browsers are doing scrollIntoView() for any element with id , there is a dedicated behaviour for ). So, answer is yes here, page arguments should be passed via querystring. Event is window.beforeunload , yes, page reload when javascript:void(location.search='some') has been set There are no surprises, have a look Also, on working with querystring: http://xkr.us/js/querystring
