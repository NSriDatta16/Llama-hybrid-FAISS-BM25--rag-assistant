In the case of , the server won't encode the image. The browser will ask for it on a separate request , and the server will just dump a short header, then file's data on the wire. In a well-written HTTP server, this is a blazingly fast operation, since there's only a minimal amount of processing involved. The browser can also cache the image so it doesn't have to retrieve it later (as others have said). Embedding it inline will both increase the weight of the page, and make it impossible to cache the image separately.
