SSIS is probably not the right choice if you are simply deleting records. This might be of interest: Performing fast SQL Server delete operations UPDATE: as Remus correctly points out, SSIS can perform well or badly depending on how the flows are written, and there have been some huge benchmarks (on high end systems). But for just deletes there are simply ways, such as a SQL Agent job running a TSQL delete in batches.
