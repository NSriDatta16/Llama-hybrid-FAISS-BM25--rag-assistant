I'm not sure how the performance rates with other options, but when we needed that type of feature we created a full-text catalog. You can find this feature in SSMS under [DatabaseName] - Storage - Full Text Catalogs. Define which fields you want to index and then let it build. Then you can query against the index. Here's a good article to supplement my terrible description: http://www.developer.com/db/article.php/3446891/Understanding-SQL-Server-Full-Text-Indexing.htm
