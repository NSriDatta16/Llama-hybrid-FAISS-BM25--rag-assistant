It turns out that in the DomainService class one has to implement (or at least to mark "placeholder methods") as "Edit", "Delete",... eg [Delete] public void DeleteCulture(Culture currentCulture) { throw new NotImplementedException("UpdateCulture not Implemented yet"); } [Insert] public void InsertCulture(Culture newCulture) { throw new NotImplementedException("InsertCulture not Implemented yet"); } This way the OrganizationDomainContextEntityContainer class creates an EntitySet with parameter EntitySetOperations.All (meaning that all the CUD operations are available). Hope it's useful for someone in the future!
