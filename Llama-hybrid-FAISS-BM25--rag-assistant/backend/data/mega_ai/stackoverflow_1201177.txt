You can do this using a common file such as include.py , but it goes against recommended practices because it involves a wildcard import. Consider the following files: app/ __init__.py foo/ a.py b.py c.py include.py Now, in a.py , etc., do: from include import * As stated above, it's not recommended because wildcard-imports are discouraged.
