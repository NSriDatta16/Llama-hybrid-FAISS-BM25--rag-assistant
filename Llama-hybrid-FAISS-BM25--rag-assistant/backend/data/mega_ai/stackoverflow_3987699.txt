My final solution was to reorganise the where clauses to ensure that the select returns only the correct datatype (IQueryable). Without reorganising the where clauses, the join data is still passed through the object despite specifying select t . public IQueryable GetData(string DataType) { IQueryable dbData = ( from t in db.All ().Where(e => e.Category == TransType) join e in WebHelpers.LocalList on t.Type equals e.Type orderby t.DateOccurred descending select t ); return dbData; }
