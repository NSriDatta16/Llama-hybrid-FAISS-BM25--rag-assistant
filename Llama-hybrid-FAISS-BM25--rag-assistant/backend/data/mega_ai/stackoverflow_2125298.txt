You can also see package names as reversed internet domain names (which is often also true in real world, see for example the org.apache.commons which correlate with http://commons.apache.org ). The com (commercial) and org (organization) are here then actually Top Level Domain names. Package names are in general just to identify the manfacturer/vendor of the code you're facing.
