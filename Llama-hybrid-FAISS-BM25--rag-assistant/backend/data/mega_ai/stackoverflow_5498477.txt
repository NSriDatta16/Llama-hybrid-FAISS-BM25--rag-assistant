Okay, solved. Problem was jnetpcap depends on libjnetpcap.so not libpcap.so. My mistake. Added the path correct path (/stuff/ ... /libjnetpcap-1.3.b4) to the Native Library Path Suffix field of the JVM Path Settings, and it was all systems go.
