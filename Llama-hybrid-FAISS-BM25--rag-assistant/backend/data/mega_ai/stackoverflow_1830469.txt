3/2 is likely chosen as "something that divides cleanly but is less than phi ". There was an epic thread on comp.lang.c++.moderated back in November 2003 exploring how phi establishes an upper bound on reusing previously-allocated storage during reallocation for a first-fit allocator. See post #7 from Andrew Koenig for the first mention of phi 's application to this problem.
