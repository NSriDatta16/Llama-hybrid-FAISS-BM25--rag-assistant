Would it work to create a simple map between the user's organization and the appropriate database connection? It could be as simple as a Dictionary initialized in global.asax. The key would be the company and the name of a connection string defined in web.config. If you need to make this configurable, you can load the values for such a dictionary from a custom configuration section.
