No. The GCC built-ins probably make good sense to the guys who write operating systems, libc, and maybe pthreads itself, but for your average application there is no reason not to use the pthreads approach. And even if you always use GCC, some day you may want to run a static analysis tool which won't handle all the customer GCC extensions.
