Each for its own. They do not overlap. CSS is the only thing that actually makes difference in appearance. Skins or not, in the end it boils down to CSS classes and style attributes, because that is the only thing that the browser understands. Skins however allow you to globally set some properties for a type of control. It's not limited only to appearance things. Yes, you can set the CssClass and Style properties, but you can also set other properties as well. It's like a global override and default value storage.
