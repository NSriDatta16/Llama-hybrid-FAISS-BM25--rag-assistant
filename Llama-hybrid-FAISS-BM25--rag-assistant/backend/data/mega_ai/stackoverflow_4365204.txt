In order to get the confirmation, you have to cross several informations: 1) The User-Agent string for an iPad 3.2 is the following: Mozilla/5.0 (iPad; U; CPU OS 3_2 like Mac OS X; en-us) AppleWebKit/531.21.10 (KHTML, like Gecko) Version/4.0.4 Mobile/7B334b Safari/531.21.10 2) Safari 4.0.4 for iPad was released on April 2010, the 3rd (see http://en.wikipedia.org/wiki/Safari_version_history ) 3) readTransaction was commited to the WebKit trunk on September 2009, the 9th (see http://trac.webkit.org/changeset/48227 ) This is why the WebKit engine that power Safari 4.0.4 for iPad 3.2 does not support readTransaction .
