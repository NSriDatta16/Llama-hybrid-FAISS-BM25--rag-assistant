If nothing else, because it isn't legal to have both a static and non- static version of a method (probably because it's legal, if discouraged, to call a static method in a non- static context). I also feel like such a method, while useful in the context of defining a logger or whatever, could be confusing in other contexts, such as when called from an instance method.
