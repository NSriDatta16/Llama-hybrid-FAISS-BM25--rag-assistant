Your average XML Parser will create Nodes which know their Parents so you can do something like: XmlNode node = this.FindNode(filter); node.ParentNode.DeleteChild(node); This way you have split both functions but no overhead. Regarding the core of your Question: Yes, searching and deleting in one Method violates the single responsibility but performance and SRP don't mix that well in many cases so you have to decide whats more important. PS: Example is not (knowingly) related to any real language out there.
