You could consider using a distributed version control system such as Mercurial . It's as easy to use as subversion. Indeed it may be better when the repo is on a local filesystem (as opposed to a server) because one of my colleagues said that svn is deprecating FSFS storage (I can't find a reference for this). For infromation on mercurial, see this link below. I wish I never bothered with svn. http://hgbook.red-bean.com/read/
