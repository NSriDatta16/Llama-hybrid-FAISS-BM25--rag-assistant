This squarely falls in the "premature optimization" category. It is not at all guaranteed that you'll cause fragmentation, the disc may well have uncommitted clusters past the file end and often does. Perhaps more the point, there isn't anything in the .NET framework that allows you to detect or fix this. Accessing the volume's MFT requires unmanaged code and admin privileges. Defragging a disk is a normal machine maintenance task. It became automatic with Win7.
