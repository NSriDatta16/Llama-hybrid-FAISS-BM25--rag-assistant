Your use case sounds like you're better off using a database to do that rather than relying solely on PHP's available resources. The advantages to using something like MySQL instead is that it's specifically engineered with memory management in mind for such things as storage and lookup. It's really no fun constantly serializing and unserializing data just to update or change a few pieces of information.
