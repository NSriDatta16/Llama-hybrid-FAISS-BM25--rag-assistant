This falls directly under the category of "Making Wrong Code Look Wrong" . In zero-based indexing languages, such as Java or C# people are accustomed to variations on the index condition. Thus, leveraging this defacto convention would make off-by-one errors more obvious. Regarding performance: any good compiler worth its memory footprint should render such as a non-issue.
