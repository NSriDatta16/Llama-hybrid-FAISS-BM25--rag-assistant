String literals should not be freed, as they aren't dynamically allocated. The leak doesn't lie in the sample code. From C++03, § 5.13.4-1 String literals: An ordinary string literal has type “array of n const char ” and static storage duration (3.7) [...] A wide string literal has type “array of n const wchar_t ” and has static storage duration
