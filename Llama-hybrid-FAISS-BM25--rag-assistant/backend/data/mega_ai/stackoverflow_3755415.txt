In this particular example, even a function would be inappropriate, as it's basic arithmetic. However, the example serves to show you the syntax and calling conventions for blocks. Blocks themselves are more useful as callbacks or as "drag and drop code." They are a way to do delegation and code extension without having to build stateful functions or delegate classes, and without having to provide the ubiquitous void *contextInfo argument to every callback.
