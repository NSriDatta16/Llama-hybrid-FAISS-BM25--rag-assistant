IMHO that's a header interpretation problem. I'm not very familiar or fond of ASP.NET but at least in PHP you need to have these: ob_start(); (* should be equivalent to HttpContext.Current.Response.Buffer = true *) header('Content-Description: File Transfer'); header('Content-Type: application/octet-stream'); header('Content-Disposition: attachment; filename=somefile.pdf'); header('Content-Transfer-Encoding: binary'); header('Expires: 0'); header('Cache-Control: must-revalidate, post-check=0, pre-check=0'); header('Pragma: public'); //header('Content-Length: ' . filesize($file)); ob_flush(); (* should be equivalent to HttpContext.Current.Response.Flush() *) ** output the pdf contents here (* header() should be similar to HttpContext.Current.Response.AddHeader() function *) Note that setting Content-Length: together with content-disposition: attachment might fail to work in Safari and IE. Hope it helps...
