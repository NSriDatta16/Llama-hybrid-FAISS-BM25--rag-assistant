You will need to specify the configInfo attribute. The value is an '&' separated list of " os,ws,arch " triplets. Scripts are only generated for platform specific fragments if they resolve for one of the configurations being built. If configInfo is not specified, the default will be " *,*,* " which means "platform independent" (which your foo.win32.x86 doesn't match). eg: The help page is here , which may be helpful.
