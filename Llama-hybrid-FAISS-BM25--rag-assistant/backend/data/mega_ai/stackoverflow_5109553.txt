Small tables can be created as an "indexed oriented table" in which case the table and the index are the same "object" in storage. I believe a primary key is required for an IOT. Before IOTs were available it was in many cases a judgement call. Certainly, if the index was almost as big or bigger than the table it's useless unless it's being used to enforce uniqueness. another Richard Foote link You have to look at the query being run and compare it to the index. An index on (A, B) covers: select B from T where A = 3
