Question 1: It is the responsibility of the Publisher database to mark transactions for replication in its transaction log. The Log Reader Agent then monitors the Publisher's transaction log, looking for marked transactions, and copies those to the Distribution database. See: sp_replcmds and sp_repldone for some additional details. Question 2: My guess (and it's only a guess) would be that since the procedure upgrades both system and user databases, it is not automatically run during an attach operation. Question 3: You can view the details of the procedure yourself. use msdb go sp_helptext 'sys.sp_vupgrade_replication' go
