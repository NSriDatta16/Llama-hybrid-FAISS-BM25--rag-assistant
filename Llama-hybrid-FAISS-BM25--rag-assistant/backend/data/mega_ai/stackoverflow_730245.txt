I'd usually say no more than five, from the 7+/-2 rule of short term memory , and some pessimism about programmer attention spans. Note: I'd count a varargs list as one entity. If you really are constrained to construct the object in one go, you can usually collect related parameters in simple value objects and pass those into the constructor. Try to ensure that the value objects make some conceptual sense and are not just random collections of information...
