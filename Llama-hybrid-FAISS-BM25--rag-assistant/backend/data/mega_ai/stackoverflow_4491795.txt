What I would do is create both, each in a DIV; then just hide whichever is not needed. This eliminates the need for heavy DOM manipulation (as you're only doing that once, on page load, there are fewer opportunities to leak memory in certain browsers cough cough ), and is harder to accidentally mess up the app state (what with Alberta and Alabama sharing the same code and all that). This would be the initial page: USA Canada and JavaScript to go with it (jQuery used here for brevity): $(document).ready(function(){ // hide both containers on page load $('#canada_select').hide(); $('#usa_select').hide(); // create and populate both boxes: $('#canada_select').append(' ' + ' Alberta ' + ' British Columbia ...' + ' ' ); $('#usa_select').append(' ' + ' Alabama ' + ' Alaska ...' + ' ' ); }; // we'll also need handlers to show the correct list, // depending on the selected country $('#country_usa').click(function(){ // we want US states $('#canada_select').hide(); $('#usa_select').show(); }); $('#country_canada').click(function(){ // we want Canadian provinces $('#usa_select').hide(); $('#canada_select').show(); }); This should be the result: USA Canada Alabama Alaska ... Alberta British Columbia ... At the backend, process state_usa iff country =='USA'; process province_canada iff country =='Canada'.
