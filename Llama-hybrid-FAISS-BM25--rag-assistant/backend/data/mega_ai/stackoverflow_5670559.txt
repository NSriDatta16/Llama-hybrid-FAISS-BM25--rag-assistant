Do you need set the user agent string of your component to retrieve such pages. for the TIdHttp component you must set the Request.UserAgent property to a valid iPhone user agent string. see this sample IdHTTP.Request.UserAgent:='Mozilla/5.0 (iPhone; U; CPU iPhone OS 3_0 like Mac OS X; en-us) AppleWebKit/528.18 (KHTML, like Gecko) Version/4.0 Mobile/7A341 Safari/528.16';
