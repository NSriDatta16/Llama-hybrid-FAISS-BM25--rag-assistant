Yes, the results are needed for real-time processing. If it's a mobile device I support, I want to send them to another page. Because...? Of a different and simpler style/look'n'feel? Use the CSS media rule for this. Handhelds will pick the one for handheld . You can then just supply a different CSS which shrinks/rearranges sections or omits certain space-consuming sections, et cetera. No need for nasty server-side user agent sniffing here.
