You can use an Excel connection in 64bit environment. Go to the package configuration properties. Debugging -> Debugging Options -> Run64BtRuntime -> change to False In addition if you use SQL Agent go to the job step properties and then check the 32 bit runtime. note: this only applies to debugging within Visual Studio...
