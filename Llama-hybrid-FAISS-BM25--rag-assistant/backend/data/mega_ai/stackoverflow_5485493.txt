The two things that I would take into consideration are: Is persisting session data in a database necessary for my application? When migrating an old application to a new platform there is a good chance you will run into some snags. Better to change the session storage at the end of the project if there is time. If the timeline has not already been accepted and you'd just like to try and add another deliverable, this is not something that should take you very long.
