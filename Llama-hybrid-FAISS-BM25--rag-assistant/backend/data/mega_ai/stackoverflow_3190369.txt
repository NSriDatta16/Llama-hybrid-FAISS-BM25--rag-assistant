I find non-logical that you use both dataUrl and value parameters of editoptions . I verified in both 3.7.1 and 3.6.5 versions of jqGrid in grid.common.js in the createEl function there are such code fragment: switch (eltype) { // ... case "select" : // ... if(typeof(options.dataUrl) != "undefined") { // ... } else if(options.value) { // ... } break; So if you define dataUrl then value parameters of editoptions will be ignored. UPDATED : The usage of formatter:'select' is not important for the question which you asked. If you send back in the response on dataUrl the values 1, 2 or 3 (the keys) instead of the values 'ABC', 'CDE' and 'EFG' then you should use formatter:'select' . If all cases the value parameter of editoptions will be ignored if you use also dataUrl .
