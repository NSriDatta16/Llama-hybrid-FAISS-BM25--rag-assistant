That's not strange output, that's valid XML. When you have an empty tag, XML lets you use a short closing syntax that's not always valid in HTML or XHTML The html_entity_decode(); function converts html entities, such as &gt; converts to > You'll need to post-process your xml fragment and convert it into proper HTML. The easiest way to do this is with the DomDocument API. $foo = new DomDocument(); $foo->loadHtml(' Testing '); echo $foo->saveHtml(); This will take an XML fragment, and convert it into and HTML document, which includes fixing all the self closing tags. You'll still need to parse out stuff in the , but that's a lot easier than fixing all the self closing tags yourself.
