Make sure that you are using the InnoDB storage engine when creating the table. Other storage engines will simply ignore foreign key constraints. ( Source ) Example: CREATE TABLE a ( id INT AUTO_INCREMENT PRIMARY KEY ) ENGINE=INNODB; CREATE TABLE b ( id INT AUTO_INCREMENT PRIMARY KEY, a_id INT, FOREIGN KEY (a_id) REFERENCES a(id) ) ENGINE=INNODB; INSERT INTO b (id, a_id) VALUES(NULL, 1); The above insert fails with: ERROR 1452 (23000): Cannot add or update a child row: a foreign key constraint fails...
