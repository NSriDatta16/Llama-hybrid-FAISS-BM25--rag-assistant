Taken literally, it is impossible. App1 and App2 will be running on different virtual machines, probably in different processes. There is no way to transport an object between them, any more than you can transport a Java object between an applet and a JavaEE server. Your options: Use remote services and AIDL to implement a remote procedure call, effectively giving you "pass by reference" between apps Use Parceable and Intent extras, effectively giving you "pass by value" between apps
