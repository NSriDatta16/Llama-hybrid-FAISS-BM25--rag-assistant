Separating your code in multiple methods allows you to reuse them. It also makes your code a lot more organized. Calling a method doesn't take a lot of resource so you shouldn't worry about performance issues unless you're developing for very sensible, low resources embedded systems for example. It's generally considered good practice to have a method per logical responsibility.
