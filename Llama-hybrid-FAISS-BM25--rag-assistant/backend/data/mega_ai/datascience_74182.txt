Though I believe DT/RF will not have any challenge in a single model but I will prefer the other choice. Multiple models Pros - 1. Simple and interpretable modelling 2. You have lesser number of dimension per data count 3. Simple maintenance and troubleshooting. Every new developer will not have to waste some time in figuring out this point. 4. If they are independent and only correlated to respective Y. This can have a relation to the data domain. So keeping separate is better for future data decisions 5. Liberty to use different models for the different dataset as per the under-lying data pattern 6. Inherent loose-coupling and low cohesion Cons 1. Manage 3 code/production instance
