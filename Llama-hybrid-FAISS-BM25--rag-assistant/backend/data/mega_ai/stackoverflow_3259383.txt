But is that still true? I'm not sure it ever was. Are you thinking about CSS elements? Because those are illegal in the body. But it is usually the better choice to put Javascript code into the head or a separate script, and wrap it in a document.ready or onload event. However, in-body Javascript can have its place, for example when embedding external JavaScripts that document.write() stuff into the document. Top-modern, bleeding-edge Google Analytics relies on a segment being inserted into the very end of the body.
