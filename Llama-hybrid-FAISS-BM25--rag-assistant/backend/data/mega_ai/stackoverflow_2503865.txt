You can also use decorators to enforce the type of attributes . >>> @accepts(int, int, int) ... @returns(float) ... def average(x, y, z): ... return (x + y + z) / 2 ... >>> average(5.5, 10, 15.0) TypeWarning: 'average' method accepts (int, int, int), but was given (float, int, float) 15.25 >>> average(5, 10, 15) TypeWarning: 'average' method returns (float), but result is (int) 15 I'm not really a fan of them, but I can see their usefulness.
