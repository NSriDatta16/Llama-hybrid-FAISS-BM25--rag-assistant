Option 3 would be the best decision. The model should be responsible for all of the data values, the controller the business logic, and the view presentation. It's always a good idea to keep the views as simple as possible and avoid embedding code in them. While you could do handle this in the controller it would need to be duplicated in each controller which utilizes this model. That could create problems down the road if you needed to make a change.
