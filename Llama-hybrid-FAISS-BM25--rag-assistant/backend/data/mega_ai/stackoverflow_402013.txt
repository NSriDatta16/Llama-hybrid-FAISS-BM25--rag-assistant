The real question isn't about hard coding, but rather repetition. If you take the excellent advice found in "The Pragmatic Programmer" , simply Don't Repeat Yourself (DRY). Taking the principle of DRY, it is fine to hardcode something at any point. However, once you use that particular value again, refactor so this value is only hardcoded once.
