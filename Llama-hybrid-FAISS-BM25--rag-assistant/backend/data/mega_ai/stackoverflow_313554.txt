You might be able to leverage some of the interoperability features that are integrated into Mono 2.0 http://www.mono-project.com/Main_Page JNBridge is another possible interoperability solution: http://www.jnbridge.com/ However, a more optimal approach might be to expose your .NET code as Services - and then access them from the Java client (or through a light-weight ESB). Of course, time, budget, resources are constraints you'll have to consider.
