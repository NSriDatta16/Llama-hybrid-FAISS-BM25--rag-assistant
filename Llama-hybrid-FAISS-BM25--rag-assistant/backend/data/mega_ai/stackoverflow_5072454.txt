It sounds like you're looking for a Distributed Hash Table . Despite the name, they can be used for more than storage - they serve as a generic routing mesh, with O(log n) expected hops to route messages between any two nodes. Although the basics remain the same, there are at least a couple of distinct approaches to how to implement them; you'll probably want to checkout Kademlia and Chord as examples. Kademlia is simpler to implement but focuses on data storage and retrieval; Chord is more complex but more versatile.
