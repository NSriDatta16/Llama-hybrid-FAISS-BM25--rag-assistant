This is not possible due to how most clients implement redirection [1]: However, most existing user agent implementations treat 302 as if it were a 303 response, performing a GET on the Location field-value regardless of the original request method. So you must use a workaround (like simply calling the method post() from the RequestHandler) or forget the idea. [1] http://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html#sec10.3.2
