While it is not as elegant as Firebug, you can use Fiddler/Fiddler2. It will create a local proxy and reroute IE's traffic through it. It will then capture all web traffic that the browser is generating (including XmlHttpRequests). You can view the post/response information and other technical details. The down side is it does not link to your client side JS handlers at all, so it does not allow any type of interactive debugging (nor could it). It only lets you observe the requests. Fiddler
