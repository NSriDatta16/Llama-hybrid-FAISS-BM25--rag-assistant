Rather than loop over the tags using '//td' , search for the tags, grab only the third one, then loop over '//td' . Mechanize uses Nokogiri internally, so here's how to do it in Nokogiri-ese: html = 11-02-2011 1 1 1 1 0 0,00 DKK 0,00 0,00 DKK 12-02-2011 5 5 1 4 0 0,00 DKK 0,00 0,00 DKK 14-02-2011 1 3 1 1 0 0,00 DKK ,00 0,00 DKK EOT require 'nokogiri' require 'pp' doc = Nokogiri::HTML(html) pp doc.search('//tr')[2].search('td').map{ |n| n.text } >> ["14-02-2011", "1", "3", "1", "1", "0", "0,00 DKK", ",00", "0,00 DKK"] Use the .search('//tr')[2].search('td').map{ |n| n.text } appended to Mechanize's agent.page , like so: agent.page.search('//tr')[2].search('td').map{ |n| n.text } It's been a while since I played with Mechanize, so it might also be agent.page.parser... . EDIT: there will come more rows in the table. The row that i want to scrape is always the second last. It's important to put that information into your original question. The more accurate your question, the more accurate our answers.
