Since documents is the web root, you should be able to use the ServletContext.getRealPath(String) method. ServletContext context = servletRequest.getSession().getServletContext(); // Or if you have the servlet instead of request, use this: // ServletContext = servlet.getServletContext(); // see comment by BalusC String virtualPath = "/images/myimage.jpg"; String realPath = context.getRealPath(virtualPath); // realPath will be D:\documents\images\myimage.jpg http://download.oracle.com/javaee/5/api/javax/servlet/ServletContext.html#getRealPath(java.lang.String) Returns a String containing the real path for a given virtual path. For example, the path "/index.html" returns the absolute file path on the server's filesystem would be served by a request for " http://host/contextPath/index.html ", where contextPath is the context path of this ServletContext.. The real path returned will be in a form appropriate to the computer and operating system on which the servlet container is running, including the proper path separators. This method returns null if the servlet container cannot translate the virtual path to a real path for any reason (such as when the content is being made available from a .war archive).
