This should be fine. Using // for protocol relative URLs is standards-compliant and supported by all browsers. According to this blog post , there are two minor IE specific downsides: Resources get (obviously) cached twice if called through http and https; and CSS style sheets get loaded twice . Both not really a problem for the average-sized site, IMO.
