Well the whole purpose of some kind of symbolic scenario really just boils down to GUI based development, which is a subject Visual Studio slowly flirts with, but we're still many years away from. Creating some kind of symbol that represents a FOR loop wouldn't speed up development. If you want to code faster just use drag'n'drop code blocks that any decent IDE already supports.
