That topic is about how to access a constant using EL. What I'm asking is where I go wrong when using an expression inside EL. Scriptlets and EL do not run in the same context. While there are workarounds/hacks, the consensus is that you should not mix them. Use the one or the other. Since scriptlets are discouraged since a decade, EL is the way to go. I'd suggest to use an enum instead. Even though you cannot reference them directly, you can just represent them as strings. public class Foo { public enum Display { YES, NO } private Display display; // ... } with ... See also: Testing enums in JSP EL
