The winning answer to this similar question suggests that you only store the user ID in the cookie. The rest goes in the database. If you can consider other approaches besides cookies, many folks prefer using session over using cookies. For one thing, you don't always have a lot of room in a cookie. Storing the shopping cart in a cookie means that you will have no record of what people were shopping for but didn't purchase. OTOH, using the cookie is using the shoppers' storage space and preserving your own. That could be significant over time and a lot of shoppers.
