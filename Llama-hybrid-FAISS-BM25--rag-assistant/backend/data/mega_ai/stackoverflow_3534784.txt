Once the compiler gets done with that code, the IL/machine code for both fragments will be identical. I tend to use string.Empty when an empty string is an important part of the algorithm. I tend to use "" for values that could be anything but for now it just happens to be an empty string. In other words, use the one that communicates the intent of the expression.
