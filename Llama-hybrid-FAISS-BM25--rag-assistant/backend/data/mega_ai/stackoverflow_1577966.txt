MVC is a design pattern. Its purpose is to separate business logic and presentation details. ASP.Net MVC is a mechanism to create web applications using ASP.Net and the MVC pattern. One of the features of ASP.NET MVC is the ability to use SEO friendly URLs to provide commands to the controller part. You can do as you have stated but ASP.Net have provided you a mechanism to do this easier. The way ASP.Net Webforms was designed is that it made it easy for you drag controls on to the web form and code the logic underneath. ASP.Net MVC is designed so you separate your concerns easier.
