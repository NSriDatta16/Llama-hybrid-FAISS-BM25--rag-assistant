Cheap option #1 : create the datatable as a local static variable to your web service class. Populate the datatable object in a static constructor to the class. The datatable object will be available to your webmethod when the service is queried for the first time as well as any app worker process recycles. //pseudo-code class MyWebMethods { private static DataTable localDataTable; static MyWebMethods() { localDataTable = new DataTable(); //Do whatever else is needed here to populate the table } [WebMethod] public string GetSomeData(int id) { //Query localDataTable using parameter info //return result } } Cheap option #2 : similar approach, consider ASP.Net Cache (System.Web.Caching.Cache). Same loading process as above, but different storage.
