Usual test methodologies applies; white box and black box. White box testing for you may mean instrumenting your application to be able to make it go into a particular state, then you can predict the the result you expect. Black box may mean that you hit a page, then consider of the possible outcomes valid. Repeat and rinse till you get sufficient coverage. Another thing we use is monitoring statistics for our service. Did we get the expected number of hits on this page. We routinely run a/b tests, and I have run a/b tests against refactored code to verify that nothing changed before rolling things out. /Allan
