From the documentation : For InnoDB tables, OPTIMIZE TABLE is mapped to ALTER TABLE, which rebuilds the table to update index statistics and free unused space in the clustered index. Beginning with MySQL 5.1.27, this is displayed in the output of OPTIMIZE TABLE when you run it on an InnoDB table, as shown here: mysql> OPTIMIZE TABLE foo; +----------+----------+----------+----------------------------------------------------------- | Table | Op | Msg_type | Msg_text +----------+----------+----------+----------------------------------------------------------- | test.foo | optimize | note | Table does not support optimize, doing recreate + analyze ... | test.foo | optimize | status | OK +----------+----------+----------+----------------------------------------------------------- You can make OPTIMIZE TABLE work on other storage engines by starting mysqld with the --skip-new or --safe-mode option. In this case, OPTIMIZE TABLE is just mapped to ALTER TABLE.
