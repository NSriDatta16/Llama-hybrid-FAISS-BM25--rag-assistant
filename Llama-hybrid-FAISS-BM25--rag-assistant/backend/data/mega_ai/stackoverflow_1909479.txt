The answer, as shown in this thread on the TeamCity support forum , is to tell MSBuild to use a special TeamCity log listener using the "/l" switch: msbuild /l:JetBrains.BuildServer.MSBuildLoggers.MSBuildLogger, The dll ships in the TeamCity agent directory: {agent}/plugins/dotnetplugin/bin/JetBrains.BuildServer.MSBuildLoggers.dll
