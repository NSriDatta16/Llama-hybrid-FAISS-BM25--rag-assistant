I don't it is possible to completely automate this, but you could do most of it. The problem is where would the topics come from? Extract a list of the most non-common words and phrases from each article and use those as tags. Then I would make a list of Topics and assign words and phrases which would fall within that topic and then match that to the tags. The problem is that you might get more than one topic per article. Perhaps the best way would be to use some form of Bayesian classifiers to determine which topic best describes the article. It will require that you train the system initially. This sort of technique is used on determining if an email is SPAM or not. This article might be of some help
