An application that depends on this is extremely fragile... Also a number of people consider any use of global constructors, especially in C, a major offense and grounds for not using a library. pthread_once provides a much cleaner way to get the same safe initialization without programmers using your library having to worry that you may have messed up the program's initial state before main is entered.
