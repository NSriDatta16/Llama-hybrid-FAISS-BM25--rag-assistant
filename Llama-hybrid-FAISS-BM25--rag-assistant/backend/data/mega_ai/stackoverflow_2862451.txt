If you are looking for a progress bar that updates in real time, I don't know of any 3rd party controls, but you can update a standard ProgressBar control based on work being done on a background thread using the WPF Dispatcher. I have implemented this pattern with multiple threads reporting back to multiple progress bars in a tree view simultaneously with no problems. Here are a few articles I found that walk through this approach: "WPF Multithreading Using the Background Worker and Reporting the Progress to the UI" (ElegantCode.com) "Simple WPF Progress Window with Cancellation" (blog.quantumbitdesigns.com)
