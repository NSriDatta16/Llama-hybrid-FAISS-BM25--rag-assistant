SQL Server 2005 ignores TOP 100 PERCENT by design. Try TOP 2000000000 instead. Now, I'll try and find a reference... I was at a seminar presented by Itzak Ben-Gan who mentioned it Found some... Kimberly L. Tripp "TOP 100 Percent ORDER BY Considered Harmful" In this particular case, the optimizer recognizes that TOP 100 PERCENT qualifies all rows and does not need to be computed at all.
