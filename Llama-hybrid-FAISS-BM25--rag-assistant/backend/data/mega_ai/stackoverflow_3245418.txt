1 - Check the length of the message you generate and switch to sending from a file it the length comes close to 4000 char. 2 - Inject cr/lf chars before every sentence or paragraph. 3 - Make sure that you have cr/lf before and after a tag opening (or just lf if that's what smtp server likes) 4 - Talk to whoever you have to about shortening the length of that embedded link - it's plain unsustainable 5 - Ask about any configuration options on the smtp server to let you send e-mail as fully encoded binary (requires MIME headers and Base64 encoding) 6 - Look for a more recent alternative to xp_smtp_sendmail, a .NET xp perhaps
