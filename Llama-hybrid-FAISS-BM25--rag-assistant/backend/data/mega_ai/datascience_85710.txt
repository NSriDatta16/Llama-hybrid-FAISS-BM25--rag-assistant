If you want to go hard on the numbers: A/B testing: This is very popular among ecommerce companies. Data science team deploys their new method and compare with the impact of old one It can happen that A/B testing is not possible due to variety of reasons. There exists however many more alternatives. Synthetic control method: After deployment of the data science product. A "fake" controle group is extracted from the data to understand the impact. This is one of the most famous application ( https://economics.mit.edu/files/11859 ) Counter-factual estimations. These do no necessarily create a new controle. But based on a model of various of biases, you can estimate impact of several "what-if"scenarios. There exists many packages for this. Google's CausalImpact ( https://google.github.io/CausalImpact/ ), Ubser causal ml ( https://github.com/uber/causalml ). IF you want to read more about this topic ( https://eng.uber.com/causal-inference-at-uber/ ) From any of these techniques, you can measure whatever KPI you wish. Increase in revenue, customer retention etc...
