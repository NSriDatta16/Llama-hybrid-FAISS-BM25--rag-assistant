I would suggest avoiding using RunWithElevatedPrivileges when interacting with SharePoint objects (wherever possible, like in your example). You should restrict its use to when you need to access resources that are outside SharePoint (for example a database, file share, etc.) Here is an excellent article that provides a very elegant approach to getting elevated privileges within a SharePoint context: http://solutionizing.net/2009/01/06/elegant-spsite-elevation/
