That may not be a right design or very optimized to do such logic at client side. If you are talking about a very large datasets and want to optimize the data retrieval from database, the logic could be at server side to fetch only changed data and send to client all the data and update the entire section. For server to KNOW about the client updates the client can set the edited rows ids in a hidden value or param to server request and server can use its logic to fetch those rows.
