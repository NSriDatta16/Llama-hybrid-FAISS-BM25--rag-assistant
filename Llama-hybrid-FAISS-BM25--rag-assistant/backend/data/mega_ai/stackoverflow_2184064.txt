Either of the following reasons may cause this behaviour: Code coverage information is only gathered for assemblies specified in the Test Run Configuration file used by Team Build (LocalTestRun.testrunconfig from the output above, though usually there are separate .testrunconfig files for local and server builds). Code coverage information is only gathered for assemblies containing code that is executed during a test run - check your test list.
