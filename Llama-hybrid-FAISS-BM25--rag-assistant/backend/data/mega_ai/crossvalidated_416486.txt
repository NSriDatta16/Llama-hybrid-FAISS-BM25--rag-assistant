Estimating Algorithmic Information Content With EDA

I've compiled over 6000 county-level measures from the US Census into a tsv file and need a high end estimate of its algorithmic information content. By "high end" I mean it should be apparent to skilled statisticians that compressing the file to that number of bits, including the decompression program's length, is feasible. By "apparent" I mean it should not take much effort to have confidence in the feasibility of writing the de/compression program in, say, "R". By "much effort" I mean that a number of skilled statisticians might find competing to achieve that level of compression, perhaps backed by a prize , an intriguing challenge. Clearly, exploratory data analysis techniques come into play in first-order estimates like this -- fully recognizing that the resulting first-order models will not be reliable. How might one apply exploratory data analysis to come up with such an estimate? What is a better way of estimating the algorithmic information content than this: Not being a skilled statistician, I thought Principal Component Analysis might be useful in generating a first-order model of the data, the length of said model in "R" code being added to the length of the residual errors in bits, would provide such an estimate. So I ran this "R" code: lotc The idea being that if one were to write an "R" program that reproduced the original data from the principal components together with residual errors, the length of that program + the length of the stored principal components + the length of the residual errors, would be an estimate of the algorithmic information content.
