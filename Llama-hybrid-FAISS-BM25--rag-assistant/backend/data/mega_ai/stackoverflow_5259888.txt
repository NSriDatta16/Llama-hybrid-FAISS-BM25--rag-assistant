Restore a backup of the affected database onto another server to get the old procedure's code, then copy just that code over. If you don't have a spare server, you can restore the database to a different database on the same server. (create a new database, then restore the backup to that - just pay attention that you're restoring onto your new/temporary database, then delete that database when you're finished) Red-Gate has SQL log rescue which might have helped (it's for SQL2000), but there might be other solutions available.
