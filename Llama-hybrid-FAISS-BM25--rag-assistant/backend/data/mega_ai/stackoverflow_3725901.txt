You can look into the layered architecture approach. Struts itself is based upon the MVC architectural pattern. From Wiki , ...In MVC: Models are not data access objects; however, in very simple apps that have little domain logic there is no real distinction to be made. Many applications use a persistent storage mechanism such as a database to store data. MVC does not specifically mention the data access layer because it is understood to be underneath or encapsulated by the model. So, you can comeup with you own data access layer that would work underneath your Model; Checkout A Simple Data Access Layer using Hibernate
