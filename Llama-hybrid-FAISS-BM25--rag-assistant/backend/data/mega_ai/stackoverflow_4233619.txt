Cross-thread GUI stuff usually ends in disaster. The windows libraries actively discourage it by throwing exceptions. Even if it was allowed, CreateWindowEx could not do this by default because it would be making some very big assumptions about your code (locks, thread safety, etc); and most Windows development is probably still essentially single threaded.
