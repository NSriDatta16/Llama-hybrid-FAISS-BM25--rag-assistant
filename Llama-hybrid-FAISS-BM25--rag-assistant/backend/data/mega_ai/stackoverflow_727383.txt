I would simply use the built in User Settings (My Project > Settings > Pick "User" for the scope of the setting); they are extremely easy to retrieve and update. And although I haven't done extensive testing, I believe they remain through application updates. Does your application use a database? If so, that would be another option for permanent storage. Edit If you decide to go with the User Settings option, here is a new question that addresses an issue with those settings and ClickOnce.
