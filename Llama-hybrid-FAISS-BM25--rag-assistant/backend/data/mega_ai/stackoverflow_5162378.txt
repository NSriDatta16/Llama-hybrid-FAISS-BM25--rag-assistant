Your best strategy for maintaining integrity between the two is to eliminate the secondary storage route and store the file bits in the database directly. Most RDBMS have a binary data type that will store straight binary bits. Have your application serialize/deserialize the bits to reconstitute the file contents as needed. Include a field for filename that includes the extension.
