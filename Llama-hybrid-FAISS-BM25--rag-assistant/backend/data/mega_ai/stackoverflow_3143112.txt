If it's readonly, then you can provide a DefiningQuery for the EntitySet in the Storage Model. This essentially acts as a read-only view of the store and you can define the result shape however you want. You can then either map it to an entity or a complex type (tho a complex type may be easier if it's read-only.) select ... from blah ...
