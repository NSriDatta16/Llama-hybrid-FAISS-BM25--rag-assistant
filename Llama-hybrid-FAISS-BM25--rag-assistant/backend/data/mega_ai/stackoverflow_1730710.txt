XPath: Is there a way to set a default namespace for queries?

Is there a way to set Java's XPath to have a default namespace prefix for expressons? For example, instead of: /html:html/html:head/html:title/text()", the query could be: /html/head/title/text() While using the namespace prefix works, there has to be a more elegant way. Sample code snippet of what I'm doing now: Node node = ... // DOM of a HTML document XPath xpath = XPathFactory.newInstance().newXPath(); // set to a NamespaceContext that simply returns the prefix "html" // and namespace URI ""http://www.w3.org/1999/xhtml" xpath.setNamespaceContext(new HTMLNameSpace()); String expression = "/html:html/html:head/html:title/text()"; String value = xpath.evaluate(query, expression);
