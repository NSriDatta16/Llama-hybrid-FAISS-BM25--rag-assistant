you can delete the corresponding file/s (forgot the exact details about the naming) via the LocalStorage, here's a code snippet I use (with bsaf, but didn't change much, afair) /** * Deletes the session state by deleting the file. Useful during development * when restoring to old state is not always the desired behaviour. * Pending: this is incomplete, deletes the mainframe state only. */ protected void deleteSessionState() { ApplicationContext context = getContext(); try { context.getLocalStorage().deleteFile("mainFrame.session.xml"); } catch (...) { } }
