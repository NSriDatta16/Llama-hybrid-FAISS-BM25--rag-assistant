Yes, it is dependent on the request message send by the browser, if it will send the Accept encoding to gzip, it might get it if the server support it. e.g take this http request message. GET google HTTP/1.1 Host: aajtak.intoday.in Accept: */* Accept-Charset: iso-8859-1, utf-8; q=0.7, *; q=0.7 Accept-Encoding: gzip, deflate, x-gzip, identity; q=0.9 Accept-Language: en;q=1.0,zh-cn, zh;q=0.5 User-Agent: Mozilla/5.0 (SymbianOS/9.1; U; en-us) AppleWebKit/413 (KHTML, like Gecko) Safari/413
