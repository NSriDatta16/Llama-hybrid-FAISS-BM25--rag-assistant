If we need a universal XML representation of a list, we will conclude that a kind of the following representation has to be used : ... ... ... ... ... Let's analyze this : The wrapper element is needed because we may have two or more lists as children of an element and must be able to uniquely and conveniently identify each list. The "SomeName" part serves to make a list's name unique. The element is needed to wrap each individual list item. It cannot be omitted because a list item can have a complex structure (such as being an xml fragment itself) and it would be challenging to identify it as one whole.
