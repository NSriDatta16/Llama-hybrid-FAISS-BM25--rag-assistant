That all depends on wether or not you are planning to actually use HTML5 features. There certainly is a strong case for HTML5 when building Javascript games with features like localStorage, WebSQL and Canvas. But if you don't plan on using there technologies you might as well stick with the technologies you are using right now. There is a consideration to make too since HTML5 isn't widely supported yet among browsers. You'll probably have to resort to fallback mechanisms if you want to support non-html5 capable browsers and that might not be worth it. It actually all boils down to your personal preference.
