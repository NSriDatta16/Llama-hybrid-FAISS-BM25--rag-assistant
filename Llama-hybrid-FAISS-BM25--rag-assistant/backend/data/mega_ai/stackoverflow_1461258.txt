This got considerably easier in the Magento 1.3 branch. All you need to do is add is Yourpackagename_Yourmodulename The tag should match the tag in the module you're trying to override. Mage_Downloadable is the classname prefix of the same. Yourpackagename_Yourmodulename is the classname prefix of your module. With this in place, Magento will check your module's controllers first for a match. app/code/local/Packagename/Modulename/controllers/DownloadController.php If no matching action is found, it'll fall back to the original. Be sure to checkout my Magento Controller Dispatch Logging article for help debugging routing issues. What's awesome about this approach (as opposed to the rewrite approach ) is you Don't need to manually require the old controller file in your controller You don't need to provide new layout rules. Magento seems to magically keep the layout handles as they were
