A likely problem is that launchd is not executing your AppleScript in the logged-in user's GUI context, and therefore AppleScript can't talk to the Finder. Make sure the plist is installed as a LaunchAgent, and not a LaunchDaemon (the plist should be located in /Library/LauchAgents or ~/Library/LaunchAgents). Try adding the following to the plist, to make the script run in the GUI context: LimitLoadToSessionType Aqua Note that this will only work reliably on 10.5 and above; I was not able to get per-user LaunchAgents working correctly on 10.4.
