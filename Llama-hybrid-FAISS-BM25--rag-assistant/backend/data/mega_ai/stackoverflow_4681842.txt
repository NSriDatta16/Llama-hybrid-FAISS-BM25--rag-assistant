Veera Sundar has written two articles on how to use Log4j 's Mapped Diagnostic Context (source code for Servlet Filters) http://veerasundar.com/blog/2009/11/log4j-mdc-mapped-diagnostic-context-example-code/ that might be possible modify for your use case. Let your application classes (with the main method) add its class name as a variable which can be written to the log files. Is it less work than adding a property at runtime? No, not really but, more elegant
