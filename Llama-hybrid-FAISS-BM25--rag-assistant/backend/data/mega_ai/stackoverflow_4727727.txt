To Elias551: LockWindowUpdate is probably not the best way to handle this since it is intended for drag and drop operations and can introduce subtle bugs when misused. See http://blogs.msdn.com/b/oldnewthing/archive/2007/02/22/1742084.aspx Instead use SendMessage(hwnd, WM_SETREDRAW, FALSE, 0)
