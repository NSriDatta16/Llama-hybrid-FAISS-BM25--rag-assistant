The answers suggesting an empty loop shall only bring dual core CPU to 50%, quad-core to 25% etc. So if that is an issue one can use something like void main(void) { omp_set_dynamic(0); // In most implemetations omp_get_num_procs() return #cores omp_set_num_threads(omp_get_num_procs()); #pragma omp parallel for for(;;) {} }
