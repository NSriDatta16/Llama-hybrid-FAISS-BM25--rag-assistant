As an alternative to OSGi (which is a good way to do it) you can use Java Web Start., if your dependencies are statical one (that's to say if you won't change the library version during runtime). If you're in that case, it's as simple as having the JNLP file generated dynamically, with the libraries version chosen according to user parameters.
