from "Pragmatic - Agile Web Development with rails 3rd edition", page 380 There’s also a less obvious constraint (with STI). The attribute type is also the name of a built-in Ruby method, so accessing it directly to set or change the type of a row may result in strange Ruby messages. Instead, access it implicitly by creating objects of the appropriate class, or access it via the model object’s indexing interface, using something such as this: person[:type] = 'Manager' man, this book really rocks
