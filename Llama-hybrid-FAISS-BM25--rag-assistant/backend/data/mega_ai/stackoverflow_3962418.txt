If you were using SQL Server 2008 then you could leverage the new FileStream feature that allows you to store binary large object (BLOB) data as files on the file system without breaking the bond with the row. Moreover, FileStream is also supported by backup and replication processes. Here is an article about FileStream feature. With SQL Server 2005, you have to do it manually.
