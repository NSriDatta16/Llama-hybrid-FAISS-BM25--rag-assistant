You want to implement your own ISessionIDManager which "creates and validates session-identifier values, and manages the storage of a session identifier in the HTTP response as well as the retrieval of a session-identifier value from the HTTP request." This is not the same as a SessionStateStoreProviderBase which is the actual implementation for a data store.
