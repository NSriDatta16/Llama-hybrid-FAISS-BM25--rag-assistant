LogAndEat can reference private fields within the function where it's defined. So: private bool caughtException; Action logAndEat = ex => { caughtException = true; }; try { // Call to a WebService } catch (SoapException ex) { logAndEat(ex); } catch (HttpException ex) { logAndEat(ex); } catch (WebException ex) { logAndEat(ex); } if (caughtException) { Console.Writeline("Ma, I caught an exception!"); } This is a trite example (!) but this potentially can be a lot tidier than passing a bunch of parameters through to a private method.
