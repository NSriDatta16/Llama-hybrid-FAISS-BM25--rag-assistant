I don't know why a separate query is made for the second reference (Description). But it is possible to extend fetching strategies via criterias to load a product with all TextRefs and TextRefItems in one shot: var criteria = session.CreateCriteria(typeof(Product)) .SetFetchMode("Description.Values", FetchMode.Join); criteria.SetResultTransformer(new DistinctRootEntityResultTransformer()); criteria.Add(Restrictions.Eq("ProdId", 1)); var list = criteria.List ();
