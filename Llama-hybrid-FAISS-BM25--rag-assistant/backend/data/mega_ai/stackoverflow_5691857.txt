The fact that unit test snippets can easily be executed in F# Interactive is another possible advantage of testing with F#. That said, it sounds like for this particular scenario/organization, it might be enough work trying to get anyone to write/execute unit tests, so it may be better to just start with some C# unit tests and focus on getting the 'unit tests are good/useful' culture instilled first.
