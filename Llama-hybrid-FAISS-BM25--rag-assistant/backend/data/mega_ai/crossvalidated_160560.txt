Something similar to k-nearest-neighbor, as you said, would be a reasonable way to approach it. Without running a k-nearest-neighbor algorithm explicitly, you can easily calculate the Euclidian (or Mahalanobis, Manhattan, Minowski, or Hamming) distance between the new organization and each of the organizations already in the data set and choose the N organizations with the smallest distance.
