IIS ought to be setting the last modified header and/or the etag based on the file itself, so if it's still caching it's possible that the browser is misbehaving, in which case you would be looking to set the No-Cache pragma. References: Microsoft KB reference , Section 14.9 of the HTTP1.1 spec Elalfer's solution 2 would probably achieve a No-Cache pragma as a by-product.
