It really depends on a combination of the organisation size, system size, importance of the system & impact of the bug eg: One Man Shop or Low Impact System (quickest - App#1 above) Time to fix bug = time to find bug + time to code fix + time to deploy to production Large Organisation or Important System (longest - App#2 above) Time to fix bug = time to find bug + time to document & prioritise bug + time to estimate cost + time to approve work on fix + time to design fix + time to document fix + time to code fix + time to document test plan + time to test fix + time to regression test + time to performance/load test + time to schedule & approve deployment + time to deploy fix Edit : How many Microsoft employees does it take to change a lightbulb? is an interesting read on the topic. 1 : See http://blogs.msdn.com/ericlippert/archive/2003/10/28/53298.aspx
