No, a named anchor or other element would need to be present on the page with the same id or name as supplied in the hash value of the url. http://en.wikipedia.org/wiki/Fragment_identifier#Examples RFC 5147 proposes something similar for plain text documents only. However, none of the mainstream browsers have implemented this functionality yet. RFC 5147 proposes a fragment identifier for text/plain [read: not HTML] documents, based on character and line positions and ranges within the document using the keywords "char" and "line".[6] This example identifies lines 11 through 20 of a text document, for instance: http://example.com/document.txt#line=10,20
