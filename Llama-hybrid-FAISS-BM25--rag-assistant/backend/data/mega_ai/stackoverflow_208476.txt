I found the iterator idiom elegant, because it has a test for more elements (ommited null/empty test for brevity): public static String convert(List list) { String res = ""; for (Iterator iterator = list.iterator(); iterator.hasNext();) { res += iterator.next() + (iterator.hasNext() ? "," : ""); } return res; }
