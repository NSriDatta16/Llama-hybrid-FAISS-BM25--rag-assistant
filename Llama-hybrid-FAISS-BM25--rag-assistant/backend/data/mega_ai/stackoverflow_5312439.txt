Yes, this is certainly possible with TeamCity. If you've already done this with CC.NET and MSBuild, it should be straightforward to call your MSBuild script from TeamCity. TeamCity supports MSBuild out of the box . BTW, the term that JetBrains uses for what you are talking about is "configurations". "Build agents" are for supporting multiple/simultaneous builds .
