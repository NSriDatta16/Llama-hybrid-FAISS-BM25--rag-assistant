A more elegant holder class could be reused for any type of object by utilizing generics: public class Holder { T item; public Holder( T item) { set( item);} public T get( ) { return item;} public void set( T item) { this.item = item;} } DoStuff( ) would then look like: public static void DoStuff( ) { String msg = ""; Holder holder = new Holder ( msg); if ( getMessage( holder)) { System.out.println( holder.get( ));} } and GetMessage( ) would have to call holder.set( a_string) .
