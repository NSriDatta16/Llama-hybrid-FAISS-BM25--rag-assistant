Why does a user needs privileges over his own schema to create packages and triggers They do not. Their own schema implies the ownership (or namespace) defined by username.object All a user needs to create a trigger in their own schema is: a table in their own schema the create trigger privilege All a user needs to create a package in their own schema is: the create package privilege If the user wants to create objects in another schema or that references other objects, or that requires storage in a tablespace then they will need permissions on those foreign objects.
