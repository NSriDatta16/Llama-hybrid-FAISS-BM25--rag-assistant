Creating another (more pragmatic) answer, to separate two approaches (see Kevin Day's comment): This can be done in one array pass (sort is O(log n)), without creating any new list. Just don't use Collections.max. Instead, just calculate the absolute value as you go and have a running minimum/maximum: List l = Arrays.asList(3, -5, -7, 10, -2, 4); int iterMin = Integer.MAX_VALUE; for(Integer i : l) if(Math.abs(i) If you want to preserve the original value, you can easily do that: int iterMinOrig = Integer.MAX_VALUE; for(Integer i : l) if(Math.abs(i)
