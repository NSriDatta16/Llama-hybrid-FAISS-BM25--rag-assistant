I think you may be misunderstanding what a Grails template is. Think of a template as a reusable fragment . A template is a GSP that starts with an underscore like _menu.gsp that you would typically render from within another GSP with the a tag like . It doesn't make sense to render both a view and a template at the same time. They are mutually exclusive at this point. Are you looking to implement a Layout? If so, see the docs or the grails.org explaination. Basically, your view you would have a tag in the tag of your view -- which indicates that the view will be meshed together with the main layout located in grails-app/views/layouts/main.gsp
