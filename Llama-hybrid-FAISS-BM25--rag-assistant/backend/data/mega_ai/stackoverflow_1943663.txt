Rule #1 in web security boils down to: NEVER trust the client. Remember that clients can do just about anything with the data you are sending them, and the data they send YOU: modify cookies for subsequent requests alter or add/remove other HTTP headers, spoof User Agents Specify any combination of data in GET/POST You should assume any data coming IN from HTTP to your application is a malicious, tained, evil mess, and sanitize accordingly.
