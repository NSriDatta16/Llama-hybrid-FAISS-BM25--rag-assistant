This isn't solely a .NET thing, the same problem occurs in Java quite often. Unless it's a completely obvious requirement, I vote for not using interfaces and simply refactoring when the need becomes clear. The pragmatic approach says to just do the simplest thing that could possibly work , and don't get caught in architecture astronautics.
