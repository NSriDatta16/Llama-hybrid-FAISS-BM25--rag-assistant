This is an issue that's been discussed in several places, and there are several possible work-arounds, but none of them are necessarily ideal. This paper discusses options such as closures, scalars, and a limited access hash using Tie::SecureHash, preferring the last approach. This blog argues that in perl there are times when encapsulation should be violated, although the comments bring up some negatives for doing so. You can also look into moose for your Perl 5 objects. It's written to encourage the use of encapsulated objects .
