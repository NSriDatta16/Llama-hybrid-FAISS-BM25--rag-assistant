The use the the "sysobjects" catalog view is discouraged as of SQL Server 2005 - instead, you should start using the catalog views from the "sys" schema (or the INFORMATION_SCHEMA views, if database-portability is important to you) like this: SELECT pr.name, m.definition, pr.type_desc, pr.create_date, pr.modify_date FROM sys.procedures pr INNER JOIN sys.sql_modules m ON pr.object_id = m.object_id WHERE is_ms_shipped = 0 ORDER BY pr.Name Marc
