Personally we: Dropped web config transformations completely. Setting is retrieved from cscfg. Development version of cscfg points to local development environment (that's stored in version control). While deploying to production, we supply secure credentials for production SQL Azure and storage. For sample of the settings management class that scans application settings and cloud environment for configuration values, you can check out open source Lokad.CQRS for Windows Azure project (see CloudSettingsProvider)
