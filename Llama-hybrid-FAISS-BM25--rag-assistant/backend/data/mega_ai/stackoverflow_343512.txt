With the Restlet framework, we exactly tried to provide this level of abstraction above HTTP details and to make REST concepts more concrete (many have a matching class like Component, Connector and Representation): http://www.restlet.org/ We are pragmatic coders and our users develop real-world applications with our RESTful framework (for Java and GWT). REST supporters aren't all pedantic but there is a learning curve like for every major technology.
