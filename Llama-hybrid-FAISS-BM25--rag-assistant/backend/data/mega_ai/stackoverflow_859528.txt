First, if you MUST use a cursor, and you're updating stuff, then declare the cursor with the FOR UPDATE clause. (See example below. Note that the example is NOT based on your code at all.) Having said that, there are a myriad of ways to use something other than cursors, often leveraging temporary tables. I would investigate that route in lieu of cursors. DECLARE LoopingCursor CURSOR LOCAL DYNAMIC FOR select sortorder from customfielddefinition where context=@targetContext FOR UPDATE OF sortorder
