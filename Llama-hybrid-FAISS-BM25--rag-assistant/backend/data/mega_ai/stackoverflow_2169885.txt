DATE is actually only a 3-byte column, as opposed to a normal INT column which is 4 bytes, so are actually smaller. When you do a query on a DATE field, when you're passing in an integer representation of the date (i.e where `date` It's more efficient to store the date as a DATE. For more info, look here: http://dev.mysql.com/doc/refman/5.0/en/storage-requirements.html
