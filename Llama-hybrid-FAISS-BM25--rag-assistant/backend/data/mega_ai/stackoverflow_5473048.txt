javax.swing.text.JTextField.setText(java.lang.String) accepts a String . You can get a String representation of s1 by writing this: jt.setText(Double.toString(s1)); Now, arguably jt.setText(String.valueOf(s1)); or jt.setText(s1 + ""); are better choices because neither will require a change if you refactor s1 to be of some other type. Personally, I've always found the third choice, string concatenation with the empty string, to be a bit of an inelegant hack.
