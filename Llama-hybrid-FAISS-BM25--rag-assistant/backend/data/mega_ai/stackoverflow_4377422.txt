you should not use & when you already have a pointer. Change this: // Set default condition pthread_mutex_init(&mut,NULL); // Question v // Condition for not null pthread_mutex_init(&notNull,NULL); // Question vi // Condition for not empty pthread_mutex_init(&notEmpty,NULL); // Question vi to this: // Set default condition pthread_mutex_init(mut,NULL); // Question v // Condition for not null pthread_mutex_init(notNull,NULL); // Question vi // Condition for not empty pthread_mutex_init(notEmpty,NULL); // Question vi Pay attention to & I have put away. mut is already a pointer, and making &mut is an attempt to get pointer to pointer to pthread_mutex_t
