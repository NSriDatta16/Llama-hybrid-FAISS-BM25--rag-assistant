In that case, you want to create your own 'Auth' class to extend and remove the 'singleton' design pattern that exists in Zend_Auth This is by no means complete, but you can create an instance and pass it a 'namespace'. The rest of Zend_Auth's public methods should be fine for you. setStorage(new Zend_Auth_Storage_Session($namespace)); // do other stuff } static function getInstance() { throw new Zend_Auth_Exception('I do not support getInstance'); } } Then where you want to use it, $auth = new My_Auth('CMSUser'); or $auth = new My_Auth('SiteUser');
