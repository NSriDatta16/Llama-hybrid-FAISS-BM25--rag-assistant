closing database objects should always be done in a finally block, so that they are closed regardless of whether an exception occurred or not. Connection c = null; CallableStatement cstmt = null; try { conn = getAConnectionSomewhere(); cstmt = conn.prepareCall("{call XXX"); cstmt.executeUpdate(); } finally { IOUtils.close(cstmt); IOUtils.close(conn); } here, i've taken the liberty to use IOUtils from commons-io
