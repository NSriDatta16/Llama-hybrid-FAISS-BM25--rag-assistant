You can eliminate the possibility of the result set changing by using a serializable transaction. Most databases support these; MySQL does if you use the InnoDB storage engine. Before issuing the first query issue a SET TRANSACTION ISOLATION LEVEL SERIALIZABLE; command. When you have completed all the statements, ROLLBACK; . See here for more details... http://dev.mysql.com/doc/refman/5.1/en/set-transaction.html
