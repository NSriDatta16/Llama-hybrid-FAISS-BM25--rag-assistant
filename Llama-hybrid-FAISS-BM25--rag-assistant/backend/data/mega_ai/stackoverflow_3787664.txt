We are not using stored procedures and no ORM framework also. Those would have been your real options. If you're not using them, you're out of luck. Maybe the best practice for you would be not technical but organizational. Set up a process regarding modification of the database. Changes to the database have to be discussed and agreed upon by all parties involved Every party involved should be notified about database changes in advance to have time to react to them
