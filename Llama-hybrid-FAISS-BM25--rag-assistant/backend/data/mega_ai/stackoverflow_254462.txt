Organization In a dev environment, the production copy of the objects are dbo but developers can develop in their own schemas. Then code can reference the prod copy or their changes very simply. Using aliases can make this technique even more simple. Also, a production database might support numerous systems or subsystems. You can use distinct schemas to keep those objects grouped.
