Here is a simple and very inelegant & potentially dangerous way. Backup DB Detach DB Rename Log file Attach DB New log file will be recreated Delete Renamed Log file. I'm guessing that you are not doing log backups. (Which truncate the log). My advice is to change recovery model from full to simple . This will prevent log bloat.
