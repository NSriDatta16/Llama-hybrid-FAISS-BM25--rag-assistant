performance -- usually the #pragma once implementation compiles faster, because it's intent is clear functionality -- both serve the same purpose -- to avoid more than one inclusion of a header file. The defined version has of course wider uses too, and if used carefully will allow to check in another file if the former was included (for conditional compilation for example) usability -- if portability is not an issue go with #pragma once -- however, this is a non-standard extension
