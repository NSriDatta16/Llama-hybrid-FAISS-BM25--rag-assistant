You unit test those helper methods by writing unit tests that exercise those portions of your code. You defined a public API in this class for a reason, right? Test that. If it works, the class works. Use code coverage tools to help you know whether a sufficient portion of that class is being tested, and if it's not, write more unit tests to exercise the public API and hit those corners that aren't being covered.
