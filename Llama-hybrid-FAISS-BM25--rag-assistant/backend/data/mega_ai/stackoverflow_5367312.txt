Consider it like an alias for the variable name. From the Cocoa Fundamentals Guide : The syntax for @synthesize also includes an extension that allows you to use different names for the property and its instance-variable storage. Consider, for example, the following statement: @synthesize title, directReports, role = jobDescrip; This tells the computer to synthesize accessor methods for properties title , directReports , and role , and to use the jobDescrip instance variable to back the role property.
