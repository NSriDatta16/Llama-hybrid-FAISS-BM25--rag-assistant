Indeed, the setPreferenceForKey() / getPreferenceForKey() -function pair provides some persistent storage, but the memory allocated is pretty limited and exceeding the limitations will crash your widget horribly. Here's the entire API reference for the WRT API: http://library.forum.nokia.com/index.jsp?topic=/Web_Developers_Library/GUID-7C69DDA4-16F1-4A8F-BDB2-4CB0015B4E81.html
