It depends on how the SecurityGroup looks like and how LookupApplication looks like. You could use ResultTransformer like: .SetResultTransformer(Transformers.AliasToBean ()) .List (); Granted that SecurityGroup has properties matchinig LookupAppliaction, or else you have to project those properties like: .SetProjection(NHibernate.Criterion.Projections.ProjectionList() .Add(Projections.Property("Number"), "OrderNumber") .Add(Projections.Property("CreatedOn"), "CreatedOn") .Add(Projections.Property("MemeberName"), "Name")) .SetResultTransformer(Transformers.AliasToBean ()) .List ();
