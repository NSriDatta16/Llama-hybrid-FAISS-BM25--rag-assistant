You have quite a few options. You can store sessions in a key:value storage, f.e. memcached (my personal favorite) You can store sessions in a database You can put reverse-proxy loadbalancers like in DNS and Your servers in the back. Then set it to make all requests from the same IPs go to the same servers, regardless of which loadbalancer they go through. In HAProxy this option is called balance source . Beware: if the number of node changes, the sessions can be lost. You can use the cookie or url_param features to avoid this. See the HAProxy documentation . It's worth reading, really.
