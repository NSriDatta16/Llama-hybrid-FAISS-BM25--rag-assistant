Myisam engine supports this functionality and I believe these 4 tables are not myisam type. as manual says : The MyISAM storage engine supports all four maintenance operations, so mysqlcheck can be used to perform any of them on MyISAM tables. Other storage engines do not necessarily support all operations. In such cases, an error message is displayed. For example, if test.t is a MEMORY table, an attempt to check it produces this result: For further Detail see myisamcheck You can change the engine of tables if they fit for myisam
