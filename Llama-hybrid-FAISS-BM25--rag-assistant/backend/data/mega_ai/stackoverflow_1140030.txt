There is already an accepted answer, however, pls consider the following code modification. public interface DogKernel { public List getDogs(); } public class GreyHoundKennel implements DogKernel { protected List greyHounds; public List getGreyHounds() { return this.greyHounds; } public List getDogs() { return getGreyHounds(); // no compilation error } public static void main(String[] args) { GreyHoundKennel inst = new GreyHoundKennel(); List dogs = inst.getDogs(); } } Java generics are indeed broken, but not that broken. BTW Scala fixes this in a very elegant way by providing variance handling.
