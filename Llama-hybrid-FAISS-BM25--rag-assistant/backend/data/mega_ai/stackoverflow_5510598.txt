The best piece of advice that comes to my mind is running a myisamchk on the respective table(s), paying special attention to the repair options . Be sure to make backups of the data files beforehand. Additional thoughts: Make sure you have restarted the database server after adding the data files. Are you sure you are addressing the correct table(s)? Note that on most Linux filesystems, the table name is case sensitive (as it's reflected by a file in the filesystem). So tablename != Tablename
