In increasing order of flexibility: The options object is almost certainly the easiest approach - not really designed for that kind of usage but I suspect it would be fine for your use case. On windows you could use system.filesystem to get hold of the WScript FileSystemObject which you can then use to create files and write to them. Also see the Google desktop API blog for embedding an SQLite database in your gadget (looks pretty easy).
