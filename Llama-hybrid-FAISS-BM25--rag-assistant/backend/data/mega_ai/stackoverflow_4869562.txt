AFAIK, No. a VARCHAR(1) requires 3 bytes storage (The storage size is the actual length of data entered + 2 bytes. Ref . a CHAR(1) requires 1 byte. From a storage perspective: A rule of thumb is, if it's less than or equal to 5 chars, consider using a fixed length char column. A reason to avoid varchar(1) (aside from the fact they they convey poor design reasoning, IMO) is when using Linq2SQL: LINQ to SQL and varchar(1) fields
