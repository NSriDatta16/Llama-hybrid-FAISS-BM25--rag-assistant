It's really a matter of style and what you (and those you work with) find to be clearer. In general, I personally find the structure: if( a ) someResult = doSomething(); else if( b ) someResult = doSomethingElse(); else someResult = doSomethingAnyways(); return someResult; clearer than: if( a ) return doSomething(); if( b ) return doSomethingElse(); return doSomethingAnyways();
