Information should be continuously persisted to the database if you want to preserve it in case of a server crash or other act of god. If you have a table(s) with the same schema in your catalog, I would think you could persist it to the database using a TableAdapter(s) fairly simply. This setup would also make retrieval of the cache quite simple when the server comes back up. Check out this article: http://msdn.microsoft.com/en-us/library/bz9tthwx%28VS.80%29.aspx ~md5sum~
