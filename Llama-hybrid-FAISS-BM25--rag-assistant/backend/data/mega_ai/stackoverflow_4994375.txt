Being able to place files directly onto the user's storage is usually restricted from web apps, unless your native client wrapper provides a ready made API you would have to write your own extension method. You could use the SQLite dump command to create a SQL backup (see under heading Converting An Entire Database To An ASCII Text File ): echo '.dump' | sqlite3 ex1 >ex1.dump Then just pass the entire thing as a string to executeSql .
