Do you have any jobs running on a timed basis such as backups or optimizations? Doing a full rebuild of indexes as part of the maintenance job can cause the log files to be larger than expected (among other things - that is one of the most common). Places to look in SSMS for jobs: Management - Maintenance Plans Management - Legacy (if you were using older versions of SQL at some point) SQL Server Agent - Jobs
