Why does princomp in R stats package use population sd if data is supplied but sample sd if covariance matrix is supplied?

In princomp of the R stats package, I noticed that the "scale" of the output varies based on whether the raw data or the covariance matrix is supplied. The scalings should be equivalent to the standard deviation of the variable, but I noticed that if the data itself is supplied, the population standard deviation (denominator n) is used whereas if the covariance matrix is supplied, then the sample standard deviation (denominator n-1) is used. I was wondering why this is the case? data("iris") sd_pop $scale - sapply(iris, sd) #> Sepal.Length Sepal.Width Petal.Length Petal.Width #> -0.002764836 -0.001455317 -0.005894167 -0.002545041 pca_data$ scale - sapply(iris, sd_pop) #> Sepal.Length Sepal.Width Petal.Length Petal.Width #> 0.000000e+00 -3.330669e-16 2.220446e-16 2.220446e-16 # When using covariance scale is sample sd pca_cov $scale - sapply(iris, sd) #> Sepal.Length Sepal.Width Petal.Length Petal.Width #> 0 0 0 0 pca_cov$ scale - sapply(iris, sd_pop) #> Sepal.Length Sepal.Width Petal.Length Petal.Width #> 0.002764836 0.001455317 0.005894167 0.002545041 Created on 2023-12-05 with reprex v2.0.2
