Marker Interfaces are used to mark the capability of a class as implementing a specific interface at run-time. The Interface Design and .NET Type Design Guidelines - Interface Design discourage the use of marker interfaces in favour of using attributes in C#, but as @Jay Bazuzi points out, it is easier to check for marker interfaces than for attributes: o is I So instead of this: public interface IFooAssignable {} public class Foo : IFooAssignable { ... } The .NET guidelines recommended that you do this: public class FooAssignableAttribute : Attribute { ... } [FooAssignable] public class Foo { ... }
