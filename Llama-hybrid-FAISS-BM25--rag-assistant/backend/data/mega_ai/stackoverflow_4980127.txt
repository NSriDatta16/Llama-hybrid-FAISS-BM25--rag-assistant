If using a JSON data source, when adding an href attribute to the data list's attr list, it seems helpful to use the file:// protocol, e.g.: { "data" : { "icon" : "/js/_demo/folder.png", "title" : "workspace", "attr" : { "href" : "file:///home/areynolds/workspace" } }, "attr" : { "rel" : "folder", "id" : "/home/areynolds/workspace" }, "state" : "closed" } The node's link can then be dragged and dropped into an HTML input field. One could either use the file:// href as-is, or use a bit of JavaScript or jQuery to clean up the path. If using an HTML data source, using file:// in the anchor's href accomplishes the same result.
