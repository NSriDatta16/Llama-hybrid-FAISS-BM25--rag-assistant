As kanaka pointed out, yahoo's get_access_token call requires the oauth_callback, and simplegeo/oauth2 does not allow you to specify it easily. Here are a couple of workarounds: Instead of simplegeo/python-oauth2, take zbowling/python-oauth2 (which is a fork of simplegeo, but with some nice new code, and bugfixes), and then add an extra parameter: parameters={'oauth_callback': 'http://example.com/callback/'} to the Client.request call. Like so: client = Client(consumer) client.request('https://api.login.yahoo.com/oauth/v2/get_request_token', method='GET', parameters={'oauth_callback': 'http://example.com/callback/'}) If you insist on working with simplegeo/python-oauth2 then check out the detailed discussion on problems with simplegeo/oauth2 and yahoo oauth, at this page . It has some code fragments that you can use.
