There is a property of the RDL dataset called "noRows" that can display a custom message if the dataset doesn't have any rows returned. It will still send the empty report, just with a special message. If you want the subscription to not fire off the report then the only way I know of to do this is to check the row count of the select statement if your stored procedure and throw an error if the row count is zero.
