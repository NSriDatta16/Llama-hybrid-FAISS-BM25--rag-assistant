A set of Framework unit tests are going to have a large amount of overlap in startup and setup functions. Changing a few methods could easily break every test in this case. Especially and ORM. That said, tests should be grouped by functionality. Query of type X, unions, joins, DDL/schema retrieval, fetch caching, statement creation, etc...
