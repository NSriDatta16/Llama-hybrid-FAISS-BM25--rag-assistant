I figure it out myself. The key is to use SetResultTransformer() passing an object of DistinctRootEntityResultTransformer as a parameter. So the query now looks like as follows Session.CreateCriteria(typeof (Node)) .SetFetchMode( "Etype", FetchMode.Join ) .SetFetchMode( "Etype.Properties", FetchMode.Join ) .SetFetchMode( "Etype.Properties.ListValues", FetchMode.Join ) .SetResultTransformer(new DistinctRootEntityResultTransformer()); I found the answer to my questions through these links: http://www.mailinglistarchive.com/html/nhusers@googlegroups.com/2010-05/msg00512.html http://ayende.com/Blog/archive/2010/01/16/eagerly-loading-entity-associations-efficiently-with-nhibernate.aspx
