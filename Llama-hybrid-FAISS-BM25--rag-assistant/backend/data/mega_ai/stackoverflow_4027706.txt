If your'e talking about techniques, I'd be careful to say that doing the above is okay. Writing C++ programs using C-style program organization techniques will likely result in lots of maintainability issues. In general, the programmer is ignoring many of the benefits that an object-oriented language provides. Simply because much of C syntax is valid C++ and many coding techniques transfer does not mean you should do them in C++. That said, using C functions and things isn't a problem, so long as your'e careful about their use. In some cases, you have to.
