You could use a separate Thread class MyThread extends Thread { public void run() { while (!interrupted()) { try { // move my object, then sleep for a millisecond sleep(1); } catch (InterruptedException e) { } } } } However, in practice, you will rarely manage to have you move function called every 1 ms because other threads are also consuming processor time. So you need to take into account the actual time between the end of the previous thread loop and the current time. I suggest you read lots of tutorials about "Game Loops", you'll learn how to organise the functions moving objects, rendering, ... This one is an interesting article . Made for Android but can be applied to standard Java.
