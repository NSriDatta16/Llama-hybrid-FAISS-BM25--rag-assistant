I would go for elegance a different way: implement your file-reading and filtering operations as generators, You'll write more lines of code, but it will be more flexible, maintainable, and performant code. See David M. Beazley's Generator Tricks for Systems Programmers , which is a really important thing for anyone who's writing this kind of code to read.
