It's a class loader issue. You're instrumenting a class (javax.swing.JComponent) managed by the bootstrap class loader, and having it refer to a class (org.leplus.sfn.tracer.ComponentTracer) managed by the system class loader. If you put your ComponentTracer class in the bootstrap class loader, the problem should disappear. java -Xbootclasspath/p: -javaagent:...
