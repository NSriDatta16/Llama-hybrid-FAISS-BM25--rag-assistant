Better yet, just make an extension method on Object instead of IEnumerable : public static T LockAndExecute (this Object obj, Func operation) { lock(obj) { return operation(); } } That way not only is your case elegant Map map = _maps.LockAndExecute(() => _maps.First(i => i.ID == arg.MapID); but you can reuse this regardless of what you're doing within the lock.
