The values inside the XML tags are called tag values. Your second example is an example of how NOT to structure XML (in general, for any data storage, XML included, you don't want to bundle >1 attribute value into one blob, unless you are forever guaranteed that the 2 values will never be used/queried separately and that performance benefits of bundling are significant). It should instead be Gaussian 5 The choice between the two is sometimes blurred, but a very good set of guidelines exists from IBM
