Anybody has the solution to this error? The root cause of the problem is that the BOOK table doesn't get generated (so EclipseLink can't map the Book entity appropriately and then doesn't recognize it as an entity). This happens because the fully qualified name of the class listed in your persistence.xml doesn't match the actual FQN ( com.apress.javaee6.chapter02.Book vs com.apress.javaee6.Book ). So either change the package of the class or the fix the persistence.xml : org.eclipse.persistence.jpa.PersistenceProvider com.apress.javaee6.Book And running your mvn exec:java -Dexec.mainClass="com.apress.javaee6.Main" will succeed: mvn exec:java -Dexec.mainClass="com.apress.javaee6.Main" -e + Error stacktraces are turned on.
