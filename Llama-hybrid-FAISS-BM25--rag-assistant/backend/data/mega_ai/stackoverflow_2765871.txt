We ended up using the sp_start_job with the name of the merge replication publication. The only downside that we found was that the name of the sql job is dynamically generated when the publication is created so if the publication is drop and recreated, then the name will change. Other than that using sp_start_job has worked beautifully. Couple other things: The sql user was added to SQLAgentOperatorRole to allow the call to sp_start_job sp_help_job was used to indicate if the job completed successfully before proceeding
