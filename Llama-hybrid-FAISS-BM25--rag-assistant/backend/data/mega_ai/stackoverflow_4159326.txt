You've hit upon one correct usage of a table: tabular data . If you've got a table of users and settings, it is perfectly correct encouraged to use a table, as it is semantically appropriate. The difficulty may lie in how you want to mark up your form. You're likely going to want to put the table in a form element so that you can place your input elements anywhere within the table.
