For katakana, Wikipedia shows you the Unicode ordering. So if you wanted to use a regex character class that caught all the katakana, I suppose you could do something like this: user> (re-seq #"[\u30a0-\u30ff]+" "日本語の文章にはスペースが必要ないって、本当？") ("スペース") Hiragana, for what it's worth: user> (re-seq #"[\u3040-\u309f]+" "日本語の文章にはスペースが必要ないって、本当？") ("の" "には" "が" "ないって") I'd be pretty amazed if any regex could detect Japanese word breaks.
