First of all you need to account for unstable connectivity - the program should be ready to deal with temporary and permanent network coverage losses. Resources consumption (memory footprint, network bandwidth and CPU time) is also to be considered. Noone wants a program that drains the battery in an hour, occupies all memory or causes a giant bill to the owner.
