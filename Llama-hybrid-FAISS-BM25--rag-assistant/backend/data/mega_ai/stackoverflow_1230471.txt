Whatever account running the job will potentially have access to run commands from the command line - so you need to think about how it will be running and what permissions the account will have. For example, if a user could create a job that would run under the context of your sqlagent and your sql agent was overpriviledged (rights to change security), she could grant herself elevated privs or hurt your machine.
