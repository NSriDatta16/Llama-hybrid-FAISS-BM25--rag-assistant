you should read this faq: Hibernate does not return distinct results for a query with outer join fetching enabled for a collection (even if I use the distinct keyword)? When you specify eager loading, the resultset contains, as you noticed, 7*3 rows but in fact you only have 7 productTypes objects in memory (& 2 extra references for each). To do what you want, you can add (be aware that the underlying sql query did not change): SetResultTransformer(new DistinctRootEntityResultTransformer()) def results = ProductType.withCriteria { fetchMode("attributes", org.hibernate.FetchMode.EAGER) SetResultTransformer(new DistinctRootEntityResultTransformer()) }
