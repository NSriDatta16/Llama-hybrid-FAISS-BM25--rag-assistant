If your threads do not share the data in any way then why have them compete for access to the same list. This might be an acceptable usage of the thread local storage pattern . .NET offers the following mechanisms. Thread.SetData and Thread.GetData ThreadStaticAttribute Or perhaps if you could abstract the threads into their own individual instances of a custom class and then use instance variables instead of a static list variable.
