There is no problem with having more than one index per table, and no, one index per table isn't really a guideline. Having too many indexes is inefficient because It requires additional storage MySQL needs to determine which index to use for a particular query Edit : As per Pablo and Mark, you need to understand how your data is being accessed in order for you to build effective indices. You can then refine and reduce the indexes.
