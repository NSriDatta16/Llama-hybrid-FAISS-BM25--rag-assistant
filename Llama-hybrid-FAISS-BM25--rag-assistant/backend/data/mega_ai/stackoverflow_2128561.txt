I think a great way for a web dev to do this would be to leverage Silverlight. New, in Silverlight 4 (currently Beta), is the Notification API. http://www.silverlight.net/learn/videos/silverlight-4-beta-videos/notification-api/ http://timheuer.com/blog/archive/2009/11/22/silverlight-4-notification-window-queue-sample.aspx Your web-app/Silverlight component can ask for permission from the user to run out-of-browser, at which point you can then create the notifications/toast. If you don't have Silverlight experience, it's great stuff, and very familiar to anyone with .NET/WebForms/client-side scripting experience. Write in .NET, C#, etc...
