If you're not worried about portability For Windows platforms on semi-recent versions of MSVC you can use #pragma init_seg to force the order in which things are initialized in a module. Take a look here for Microsoft's suggestions. Note that the IBM C++ compiler and a few others also support this fashion of static initialization but not with the same syntax.
