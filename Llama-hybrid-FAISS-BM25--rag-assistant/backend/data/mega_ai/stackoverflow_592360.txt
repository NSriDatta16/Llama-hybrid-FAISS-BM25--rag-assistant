Converting theoretically portable C code into theoretically portable C code across architectures to support a hardware change that saves the company X dollars per unit. The size varies - this is a common need, and I've done small and large projects. I learned to write more portable C code. Elegance is great, but when it comes right down to it the compiler takes care of performance, and the code should be as simple and portable as possible.
